# Project documentation

This directory contains maintained documentation for the current game and its operating workflows.

- [`rules/`](rules/README.md) contains authoritative gameplay specifications.
- [`design/`](design/README.md) contains active design notes that are not yet rules contracts.
- [`deployment.md`](deployment.md) documents Railway, RunPod, Vercel, and wiki deployment boundaries.

Card data and the player-facing keyword glossary remain under [`../data/`](../data/README.md). Historical plans, completed-phase notes, and debugging evidence remain under `../.planning/`.
