<task type="auto">
  <name>Task 1: Create sync_taxonomy.py with element and tribe page generators</name>
  <files>wiki/sync/sync_taxonomy.py</files>
  <action>
Create `wiki/sync/sync_taxonomy.py` with the following functions:

**Data scanning functions:**
- `scan_elements(cards_dir: Path) -> list[str]`: Scan all `data/cards/*.json`, collect unique `element` values, return sorted list. Title-case them (e.g., "fire" -> "Fire").
- `scan_tribes(cards_dir: Path) -> list[str]`: Same for `tribe` values. These are already title-cased in the JSON.

**Wikitext generation (pure functions, no wiki connection):**
- `element_page_wikitext(element: str) -> str`: Generate wikitext for an element page. Content structure:
  ```
  '''Fire''' is one of seven elements in [[Grid Tactics TCG]].

  == Cards ==
  {{#ask:
   [[Category:Card]]
   [[Element::Fire]]
   |?Cost
   |?Attack
   |?HP
   |?Tribe
   |format=broadtable
   |limit=50
   |sort=Name
  }}

  [[Category:Element]]
  ```
- `tribe_page_wikitext(tribe: str) -> str`: Generate wikitext for a tribe page. Content structure:
  ```
  '''Rat''' is a tribe in [[Grid Tactics TCG]].

  == Members ==
  {{#ask:
   [[Category:Card]]
   [[Tribe::Rat]]
   |?Cost
   |?Element
   |?Attack
   |?HP
   |format=broadtable
   |limit=50
   |sort=Name
  }}

  [[Category:Tribe]]
  ```

**Upsert function:**
- `upsert_taxonomy_pages(site, pages: dict[str, str], dry_run: bool = False) -> dict[str, int]`: Generic upsert that takes a dict of `{page_title: wikitext}` and upserts each. Returns counts dict `{"created": N, "updated": N, "unchanged": N}`. Use the same `rstrip()` comparison pattern from `sync_wiki.py` for idempotency.

**Orchestration functions:**
- `sync_elements(site, cards_dir: Path, dry_run: bool = False) -> dict`: Scan elements, generate wikitext, upsert pages. Page titles are just the element name (e.g., "Fire", "Wood") -- NOT namespaced.
- `sync_tribes(site, cards_dir: Path, dry_run: bool = False) -> dict`: Same for tribes. Page titles are the tribe name (e.g., "Rat", "Knight").

Use the same repo root detection pattern as sync_cards.py: `_REPO_ROOT = Path(__file__).resolve().parent.parent.parent`.

Import pattern: `from sync.client import get_site` (only used if the module is run standalone; the CLI in sync_wiki.py passes the site object).
