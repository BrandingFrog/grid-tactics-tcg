<task type="auto">
  <name>Task 1: Create sync_polish.py with mobile CSS, logo/favicon, and search verification</name>
  <files>wiki/sync/sync_polish.py</files>
  <action>
Create `wiki/sync/sync_polish.py` with the following functions:

1. **`push_mobile_css(site, dry_run=False)`** — Appends responsive CSS rules to `MediaWiki:Common.css`. The CSS should:
   - Read the current page content first (preserve existing dark theme CSS)
   - Only append if the responsive block is not already present (idempotent — check for a marker comment like `/* --- Grid Tactics Mobile --- */`)
   - Add media queries for `max-width: 480px`:
     - Card infobox: `width: 100% !important; float: none !important;`
     - SMW result tables: `overflow-x: auto; display: block;`
     - Body content: `padding: 0.5em;`
     - Navigation sidebar: collapse or hide (`.mw-sidebar { display: none; }` for Vector 2022)
     - Card art images: `max-width: 100%; height: auto;`
   - Return "created"/"updated"/"unchanged" status string.

2. **`generate_logo_png(output_path)`** — Generate a simple 135x135 PNG logo programmatically using Pillow (PIL). Design: dark background (#1a1a1a), white text "GT" in bold, with a subtle border. This avoids needing external assets. If Pillow is not available, fall back to creating a 1x1 transparent PNG placeholder and print a warning.

3. **`generate_favicon_ico(output_path)`** — Generate a 32x32 ICO file. Same "GT" design scaled down. If Pillow unavailable, create minimal valid ICO bytes.

4. **`upload_logo(site, dry_run=False)`** — Generate logo to a temp file, upload as `File:Wiki.png` via `site.upload()`. MediaWiki uses `$wgLogo` which defaults to `Wiki.png`. Return status string.

5. **`upload_favicon(site, dry_run=False)`** — Generate favicon to a temp file, upload as `File:Favicon.ico`. Return status string.

6. **`configure_logo_and_favicon(site, dry_run=False)`** — Push MediaWiki:Common.css rules to reference the uploaded logo (if Vector 2022 skin needs CSS overrides for logo placement). Also push a line to `MediaWiki:Common.js` or use `$wgFavicon` via `MediaWiki:Favicon.ico` convention. For MediaWiki 1.42 on Railway, the simplest approach is:
   - Logo: Edit `MediaWiki:Common.css` to add `.mw-logo-icon { background-image: url('/w/images/...') !important; }` — but actually, for this Taqasta/mediawiki:1.42 setup, the logo is configured via `$wgLogo` in LocalSettings.php. Since we can't edit LocalSettings.php via the API, instead push the logo to a known location and add a CSS override in Common.css that replaces the default logo. Alternatively, check if `MediaWiki:Sidebar` or `MediaWiki:Logo` interface messages exist.
   - Favicon: Add `<link rel="icon">` via `MediaWiki:Common.js` using `document.head.appendChild()`.

7. **`verify_search(site)`** — Use the MediaWiki API `action=query&list=search` to search for "Rat" and verify Rat-tribe cards appear. Search for a keyword like "Ranged" and verify cards with that keyword appear. Print results. Return True/False.

**Important notes:**
- Follow the existing pattern from `sync_homepage.py` for idempotent page edits (rstrip comparison).
- Use `mwclient` API — `site.api('query', list='search', srsearch='Rat')` for search verification.
- Install Pillow in the wiki venv if not present: `pip install Pillow`.
- For logo/favicon, use `tempfile.NamedTemporaryFile` for generation, then upload.
  </action>
