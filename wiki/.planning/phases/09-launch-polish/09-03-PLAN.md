**backup.py:**

Create `wiki/sync/backup.py` that:
1. Uses the MediaWiki API to export all wiki pages as XML via `action=query&export` (Special:Export format). This is the simplest backup approach that does NOT require direct DB access (which Railway doesn't expose easily for cron jobs).
2. Fetches all pages in key categories: Card, Element, Tribe, Keyword, Rules, Patch.
3. Also fetches special pages: Main Page, Template:Card, Template:DeprecatedCard, Template:Patch, MediaWiki:Common.css.
4. Writes the XML export to a timestamped file: `wiki_backup_YYYY-MM-DD.xml`.
5. Also exports a JSON manifest listing all backed-up page titles and their revision IDs.
6. Uses `mwclient` and reads credentials from environment (same as sync scripts).
7. Prints summary: N pages exported, output file size.
8. CLI usage: `python -m sync.backup [--output-dir DIR]`

**Why XML export instead of mysqldump:** Railway doesn't expose the MariaDB port externally, and running `mysqldump` requires SSH/exec access to the container. MediaWiki's XML export captures all page content, revision history, and is the standard MediaWiki backup format. It can be re-imported via `maintenance/importDump.php`.

**wiki-backup.yml:**

Create `.github/workflows/wiki-backup.yml`:
```yaml
name: Wiki Backup (Weekly)

on:
  schedule:
    - cron: '0 3 * * 0'  # Every Sunday at 3 AM UTC
  workflow_dispatch: {}   # Allow manual trigger

jobs:
  backup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install dependencies
        run: |
          cd wiki
          pip install mwclient python-dotenv

      - name: Run backup
        env:
          MW_API_URL: ${{ secrets.MW_API_URL }}
          MW_BOT_USER: ${{ secrets.MW_BOT_USER }}
          MW_BOT_PASS: ${{ secrets.MW_BOT_PASS }}
        run: |
          cd wiki
          python -m sync.backup --output-dir ./backup_output

      - uses: actions/upload-artifact@v4
        with:
          name: wiki-backup-${{ github.run_id }}
          path: wiki/backup_output/
          retention-days: 90
```

**Important:**
- The workflow needs MW_API_URL, MW_BOT_USER, MW_BOT_PASS as GitHub secrets (same ones used in the drift-check workflow from Phase 8).
- Use `workflow_dispatch` to allow manual triggering for testing.
- The XML export format is MediaWiki-native and can be restored with `php maintenance/importDump.php`.
