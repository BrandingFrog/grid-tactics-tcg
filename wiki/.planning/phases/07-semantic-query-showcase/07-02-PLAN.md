<task type="auto">
  <name>Task 1: Create sync_showcase.py with 7 semantic queries</name>
  <files>wiki/sync/sync_showcase.py</files>
  <action>
Create `wiki/sync/sync_showcase.py` with two functions:

1. `showcase_page_wikitext() -> str`: Returns wikitext for the `Semantic:Showcase` page. The page should have:

   - Intro paragraph: "This page demonstrates the power of Semantic MediaWiki queries against the Grid Tactics card database. Each query runs live against the wiki's semantic store -- results update automatically as cards are added or changed."

   - Section "== Query 1: Fire Minions Under 3 Mana ==" with description "Budget fire cards for aggressive early-game openings." and this ask query:
     ```
     {{#ask:
      [[Category:Card]]
      [[Element::Fire]]
      [[CardType::Minion]]
      [[Cost::<<3]]
      |?Cost
      |?Attack
      |?HP
      |?Tribe
      |format=broadtable
      |sort=Cost
     }}
     ```
     Expected: Fire Imp (1), Emberplague Rat (2).

   - Section "== Query 2: High-Attack Minions by Tribe ==" with description "All minions with attack power above 10, sorted by tribe for deckbuilding reference." and query:
     ```
     {{#ask:
      [[Category:Card]]
      [[CardType::Minion]]
      [[Attack::>>10]]
      |?Attack
      |?HP
      |?Cost
      |?Element
      |?Tribe
      |format=broadtable
      |sort=Tribe,Attack
      |order=asc,desc
      |limit=50
     }}
     ```

   - Section "== Query 3: Cards Changed in Recent Patches ==" with description "Track what's been modified recently for meta awareness." and query:
     ```
     {{#ask:
      [[Category:Card]]
      [[LastChangedPatch::+]]
      |?LastChangedPatch
      |?CardType
      |?Element
      |?Cost
      |format=broadtable
      |sort=LastChangedPatch
      |order=desc
      |limit=20
     }}
     ```
     Note: Uses `[[LastChangedPatch::+]]` (has any value) rather than hardcoding patch versions. This future-proofs the query.

   - Section "== Query 4: Tanky Minions (HP >= 30) ==" with description "The most durable minions for defensive strategies." and query:
     ```
     {{#ask:
      [[Category:Card]]
      [[CardType::Minion]]
      [[HP::>>29]]
      |?HP
      |?Attack
      |?Cost
      |?Element
      |?Tribe
      |format=broadtable
      |sort=HP
      |order=desc
      |limit=50
     }}
     ```
     Expected: Iron Guardian (50), Stone Golem (50), Fallen Paladin (42), Flame Wyrm (40), Holy Paladin (40), etc.

   - Section "== Query 5: Low-Cost React Cards ==" with description "Cheap responses to keep in hand for the react window." and query:
     ```
     {{#ask:
      [[Category:Card]]
      [[CardType::React]]
      |?Cost
      |?Element
      |?RulesText
      |format=broadtable
      |sort=Cost
     }}
     ```
     Expected: Dark Mirror (1), Shield Block (1), Counter Spell (2).

   - Section "== Query 6: Dark Element Cards ==" with description "The full roster of Dark-aligned cards across all types." and query:
     ```
     {{#ask:
      [[Category:Card]]
      [[Element::Dark]]
      |?CardType
      |?Cost
      |?Attack
      |?HP
      |?Tribe
      |format=broadtable
      |sort=CardType,Cost
      |limit=50
     }}
     ```
     Expected: 7 cards (Dark Assassin, Dark Sentinel, Grave Caller, Ratchanter, Shadow Knight, Shadow Stalker + Dark Drain, Dark Matter Infusion, Dark Mirror).

   - Section "== Query 7: Metal Robots ==" with description "The full Robot tribe lineup in the Metal element." and query:
     ```
     {{#ask:
      [[Category:Card]]
      [[Element::Metal]]
      [[Tribe::Robot]]
      |?Cost
      |?Attack
      |?HP
      |format=broadtable
      |sort=Cost
     }}
     ```
     Expected: Blue/Green/Red Diodebot (2 each), RGB Lasercannon (3), Surgefed Sparkbot (5).

   - Section "== Writing Your Own Queries ==" with a brief guide: link to SMW ask documentation, list available properties from schema.py (Name, CardType, Element, Tribe, Cost, Attack, HP, Range, Keyword, LastChangedPatch, RulesText), and show a template query skeleton.

   - Footer: `[[Category:Rules]]`

2. `sync_showcase_page(site, dry_run=False) -> str`: Upserts the page titled `Semantic:Showcase` to the wiki. Same `rstrip()` idempotency pattern. Returns status string.

IMPORTANT: Use `<<` and `>>` for SMW less-than/greater-than comparisons (not `<` / `>`). These are SMW's comparison operators.
  </action>
  <verify>
