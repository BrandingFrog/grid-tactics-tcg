  <files>wiki/tests/test_idempotency.py</files>
  <action>
Create a pytest test file that mocks `mwclient.Site` and proves the idempotency contract.

The test strategy:
1. Create a mock `mwclient.Site` where `site.pages[title]` returns a mock page object.
2. On first call to `upsert_card_page()`, the mock page has `exists=False`, so status is "created" and `.edit()` is called.
3. After the first call, update the mock so `page.exists=True` and `page.text()` returns the wikitext that was just written (capture it from the `.edit()` call args).
4. On second call to `upsert_card_page()` with the SAME card data, status must be "unchanged" and `.edit()` must NOT be called again.
5. Repeat the same pattern for `upsert_taxonomy_pages` from `sync_taxonomy.py` — first call creates, second call returns all "unchanged".
6. Repeat for `sync_main_page` from `sync_homepage.py` — second call returns "unchanged".

Use `unittest.mock.MagicMock` for the site object. Load a real card JSON from `data/cards/minion_ratchanter.json` for realistic data.

Key detail: the idempotency comparison is `page.text().rstrip() == expected.rstrip()` — the mock must replicate this (store text WITHOUT trailing newline, since MediaWiki strips it).

Test class: `TestIdempotency` with methods:
- `test_card_upsert_idempotent` — second upsert_card_page returns "unchanged", zero .edit() calls
- `test_taxonomy_upsert_idempotent` — second upsert_taxonomy_pages returns all "unchanged"
- `test_homepage_upsert_idempotent` — second sync_main_page returns "unchanged"
  </action>
