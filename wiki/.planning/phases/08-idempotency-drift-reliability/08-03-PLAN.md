Create a GitHub Actions workflow at the REPO ROOT (not wiki/) that runs drift check on push to master.

**Workflow: `.github/workflows/drift-check.yml`**

```yaml
name: Wiki Drift Check

on:
  push:
    branches: [master]
    paths:
      - 'data/cards/**'
      - 'data/GLOSSARY.md'
      - 'wiki/sync/**'
      - 'src/grid_tactics/server/static/VERSION.json'
  workflow_dispatch:  # manual trigger

jobs:
  drift-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install dependencies
        working-directory: wiki
        run: |
          pip install -e .
          pip install pytest

      - name: Run unit tests
        working-directory: wiki
        run: python -m pytest tests/ -v

      - name: Check wiki drift
        working-directory: wiki
        env:
          MW_API_URL: ${{ secrets.MW_API_URL }}
          MW_BOT_USER: ${{ secrets.MW_BOT_USER }}
          MW_BOT_PASS: ${{ secrets.MW_BOT_PASS }}
          MW_API_PATH: ${{ secrets.MW_API_PATH }}
        run: python -m sync.check_drift --cards-only
```

**Key design decisions:**
- Trigger on pushes to master that touch card data, glossary, sync code, or version file.
- `workflow_dispatch` allows manual triggering from GitHub UI.
- `paths` filter avoids running on unrelated commits (RL engine changes, etc.).
- Uses `--cards-only` flag for faster CI runs (taxonomy pages rarely drift).
- Secrets `MW_API_URL`, `MW_BOT_USER`, `MW_BOT_PASS`, `MW_API_PATH` must be configured in GitHub repo settings.
- Runs tests first, then drift check (tests are fast, drift check hits live wiki).
- If drift check fails (exit 1), the workflow fails, which shows as a red X on the commit.

Create the `.github/workflows/` directory if it doesn't exist.
