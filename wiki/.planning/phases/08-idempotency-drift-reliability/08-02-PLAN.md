<task type="auto">
  <name>Task 1: Create check_drift.py — drift detection CLI</name>
  <files>wiki/sync/check_drift.py</files>
  <action>
Create `wiki/sync/check_drift.py` as a standalone module that compares live wiki page content
against the expected wikitext computed from JSON source files.

**Core function `check_drift(site, cards_dir, art_dir, glossary_path) -> list[DriftReport]`:**

1. Load all card JSONs from cards_dir, build name_map.
2. For each card, compute expected wikitext via `card_to_wikitext(card, name_map, art_exists=...)`.
3. Fetch the live wiki page `Card:{card_name}` via `site.pages[title].text()`.
4. Compare `live.rstrip()` vs `expected.rstrip()` (same idempotency pattern used everywhere).
5. If they differ, record a `DriftReport(page_title, drift_type="content_mismatch", details=difflib_unified_diff_snippet)`.
6. If page doesn't exist on wiki but should, record `DriftReport(page_title, drift_type="missing_page")`.
7. Also check taxonomy pages: elements, tribes, keywords (generate expected wikitext using existing functions from sync_taxonomy.py).
8. Also check Main Page and Semantic:Showcase page.

**DriftReport dataclass:**
```python
@dataclass
class DriftReport:
    page_title: str
    drift_type: str  # "content_mismatch", "missing_page", "extra_page"
    details: str = ""
```

**CLI (`main()`):**
- `python -m sync.check_drift` — runs full drift check, prints report, exits 0 if clean, 1 if drift found.
- `--cards-only` flag to check only card pages (faster for CI).
- `--verbose` flag to show unified diff for each mismatched page.
- Output format: one line per drifted page, e.g. `DRIFT Card:Ratchanter — content_mismatch (3 lines differ)`.
- Summary line: `Drift check: 34 pages checked, 2 drifted, 0 missing` or `Drift check: 34 pages checked, 0 issues. CLEAN.`

Import taxonomy page generation functions: `build_element_page`, `build_tribe_page`, etc. from sync_taxonomy.
Import homepage/showcase generation from sync_homepage/sync_showcase.

Use `difflib.unified_diff` for the details field (first 20 lines of diff to keep output readable).

The module must be runnable as `cd wiki && python -m sync.check_drift`.

<task type="auto">
  <name>Task 2: Add resume manifest to sync_wiki.py --all-cards</name>
  <files>wiki/sync/sync_wiki.py</files>
  <action>
Add resume-on-failure logic to the `--all-cards` path in `sync_wiki.py`.

**Resume manifest file:** `wiki/.sync_resume.json`

**On failure during --all-cards batch:**
1. When an exception occurs during the card loop (NOT a per-card error caught by the existing try/except, but an unrecoverable error like network timeout or auth failure), catch it at the batch level.
2. Write `.sync_resume.json` containing:
   ```json
   {
     "command": "all-cards",
     "started_at": "ISO timestamp",
     "failed_at": "ISO timestamp",
     "completed_card_ids": ["ratchanter", "blue_diodebot", ...],
     "remaining_card_ids": ["counter_spell", ...],
     "dry_run": false
   }
   ```
3. Print: `BATCH FAILED after N/M cards. Resume manifest written to .sync_resume.json. Rerun --all-cards to continue.`

**On startup of --all-cards:**
1. Check if `.sync_resume.json` exists.
2. If it exists, load it. Filter the cards list to only those in `remaining_card_ids`.
3. Print: `Resuming from manifest: N remaining cards (M previously completed).`
4. After successful completion, delete `.sync_resume.json`.
5. If `--no-resume` flag is passed, ignore the manifest and do a full sync (delete manifest first).

**Implementation detail:**
- Wrap the existing card loop in a try/except that catches `Exception` (but NOT KeyboardInterrupt).
- The per-card try/except inside the loop stays as-is (it handles individual card errors gracefully).
- The outer try/except catches batch-level failures (network down, auth expired, etc.).
- Add `--no-resume` flag to argparse (not mutually exclusive, just an optional flag alongside --all-cards).

Add `.sync_resume.json` to `wiki/.gitignore` (create the file if it doesn't exist, or append to it).
  </action>
