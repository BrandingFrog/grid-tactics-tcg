  <files>wiki/sync/sync_patches.py, wiki/sync/sync_wiki.py</files>
  <action>
**sync_patches.py changes:**

Add a new function `update_card_histories(site, diff: PatchDiff, repo_root: Path, dry_run: bool = False) -> list[dict]` that:

1. For each `CardChange` in `diff.cards`:
   a. **Added cards**: Read the card page `Card:{card_name}` from wiki. Parse existing history with `extract_history_section`. Append a new entry `{version: diff.version, date: diff.commit_date, change_type: "added", changed_fields: []}`. Re-render the card page: original card template invocation (from `card_to_wikitext` with `last_changed_patch=diff.version`) + `build_history_section(entries)`. Edit the page.
   
   b. **Changed cards**: Read the card page from wiki. Parse existing history with `extract_history_section`. Append new entry with `change_type: "changed"` and `changed_fields` from the CardChange. Re-render card template from the NEW JSON (at `diff.commit_sha`) using `card_to_wikitext` with `last_changed_patch=diff.version`. Combine with updated history section. Edit the page.
   
   c. **Removed cards**: Read existing card page from wiki. If it exists, wrap it with `build_deprecated_wikitext(card_name, diff.version, existing_text)`. Edit the page. Do NOT delete.

2. For changed cards, load the card JSON at the new commit using `_git_show` from patch_diff (import it). Parse JSON, build name_map from all card files at that commit, call `card_to_wikitext(card, name_map, art_exists=True, last_changed_patch=diff.version)`.

3. Return a list of result dicts `{card_name, page, status}` for reporting.

4. Call `update_card_histories` from within `sync_patch()`, right after generating the patch page wikitext and editing the patch page. The card history updates happen as part of the same sync operation.

**Important implementation details:**
- Import from card_history: `build_history_section, extract_history_section, build_deprecated_wikitext`
- Import from sync_cards: `card_to_wikitext, build_card_name_map, derive_keywords, build_rules_text`
- Import from patch_diff: `_git_show, _git_ls_tree` (for reading card JSON at specific commits)
- When re-rendering a changed card, you need to build the name_map from ALL card JSONs at the new commit SHA, not from the filesystem. Use `_git_ls_tree` + `_git_show` to load all cards at that SHA, build the name_map from them.
- For the card page text structure: card template invocation first, then blank line, then history section. The `extract_history_section` strips the old history, and we append the new one.
- Handle the case where a card page doesn't exist yet (new card added in a commit that wasn't synced card-by-card first). In this case, create the full page from scratch with the history section.

**sync_wiki.py changes:**

Add a `bootstrap_deprecated_template(site)` call that reads `wiki/sync/templates/DeprecatedCard.wiki` and upserts `Template:DeprecatedCard`. Follow the same idempotent pattern as `bootstrap_patch_template` in sync_patches.py.

3. In the `--patch` handler, call template bootstraps before syncing patches: bootstrap Template:Patch, Template:DeprecatedCard, and re-upload Template:Card if changed.
