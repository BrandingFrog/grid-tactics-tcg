  <name>Task 1: Create card_history.py module</name>
  <files>wiki/sync/card_history.py</files>
  <action>
Create `wiki/sync/card_history.py` with two pure functions (no wiki connection needed):

1. `build_history_section(history_entries: list[dict]) -> str`
   - Takes a list of history entry dicts, each with keys: `version` (str), `date` (str), `change_type` (Literal["added", "changed", "removed"]), `changed_fields` (list[str], empty for added/removed).
   - Returns wikitext for a `== History ==` section.
   - Format for each entry:
     ```
     ; [[Patch:{version}|{version}]] ({date})
     : {description}
     ```
   - Description logic:
     - "added": "Card added."
     - "changed": "Changed: {field1}, {field2}, ..." (fields joined with commas, underscores replaced with spaces)
     - "removed": "Card removed."
   - Entries sorted newest-first (reverse lexicographic by version string, matching Phase 5 convention).
   - Returns empty string if history_entries is empty.

2. `build_deprecated_wikitext(card_name: str, last_patch: str, original_card_wikitext: str) -> str`
   - Returns wikitext that:
     - Starts with `{{DeprecatedCard|patch={last_patch}}}`
     - Preserves the original card template invocation below it (so the page still renders the card's last state)
     - Adds `[[Category:Deprecated]]` at the end
   - This function is used when a card is removed from JSON -- the wiki page is preserved with the deprecated marker instead of being deleted.

3. `extract_history_section(page_text: str) -> tuple[str, list[dict]]`
   - Parses an existing card page's wikitext to extract and separate the `== History ==` section.
   - Returns a tuple: (page_text_without_history, parsed_entries)
   - parsed_entries is a list of dicts with keys: version, date, change_type, changed_fields
   - This is needed so that when we re-sync a card, we can preserve its accumulated history.
   - Parse logic: find `== History ==` header, then parse `;` definition-list lines for version/date, and `:` lines for the description. Use regex to extract version from `[[Patch:X.Y.Z|X.Y.Z]]`.
   - If no history section exists, returns (original_text, []).

All functions are pure -- no side effects, no wiki connection. Follow the same module style as patch_diff.py and patch_page.py (docstrings, type hints, __future__ annotations).
  </action>
