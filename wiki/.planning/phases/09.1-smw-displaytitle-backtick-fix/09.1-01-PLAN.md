<objective>
Phase 9.1: Fix the SMW DisplayTitleLookup backtick bug that makes every SMW-prefetched category page (starting with `/wiki/Category:Card`) return HTTP 500 on the production Railway wiki.

**Purpose:** `Category:Card` is the primary entry point from the in-app Wiki nav link and from `Main Page → All Cards`. It is also a hard prerequisite for Phase 9.2 (Semantic Drilldown) because Drilldown's landing page hits the same DisplayTitleLookup prefetch codepath. The wiki is currently unusable for category browsing.

**Root cause** (fully characterized in `09.1-RESEARCH.md`): SMW 5.1.0's `DisplayTitleLookup::fetchFromTable` pre-wraps the table name with `$connection->tablename(...)`, producing `` `smw_fpt_dtitle` ``. MW 1.43's hardened `Wikimedia\Rdbms\Platform\SQLPlatform::addIdentifierQuotes` then rejects the pre-quoted identifier and throws `InvalidArgumentException`. Upstream SMW **fixed this in PR #6172** (merged 2025-08-04, released in SMW 6.0.0 on 2025-08-12 and 6.0.1 on 2025-08-26). The fix is literally one line: drop the `tablename()` wrapper. SMW 5.1.x is flagged **obsolete** by upstream and officially only supports MW up to 1.43.1 — we run MW 1.43.8, already outside the window.

**Output:** A two-line change to `wiki/Dockerfile` (cache-bust comment bump + SMW pin `~5.0` → `~6.0`), pushed to master, deployed on Railway, verified green via live-wiki curl, followed by a Railway env-var flip (`MW_DEBUG=1` → `0`) and planning-file updates marking Phase 9.1 complete.
</objective>
