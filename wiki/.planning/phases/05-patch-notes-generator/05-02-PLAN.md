  <files>wiki/sync/sync_patches.py</files>
  <action>
Create `wiki/sync/sync_patches.py` with the following:

**State management:**

1. `_SYNC_STATE_PATH = Path(__file__).resolve().parent.parent / ".sync_state.json"`

2. `load_sync_state() -> dict`
   - Read .sync_state.json if it exists, return dict with key `last_synced_sha` (str or None)
   - If file doesn't exist, return `{"last_synced_sha": None}`

3. `save_sync_state(sha: str) -> None`
   - Write `{"last_synced_sha": sha, "last_synced_at": "<ISO timestamp>"}` to .sync_state.json

**Template bootstrap:**

4. `bootstrap_patch_template(site) -> str`
   - Import PATCH_TEMPLATE_WIKITEXT from sync.patch_page
   - Upsert `Template:Patch` on the wiki (idempotent, same rstrip comparison pattern as other bootstrap scripts)
   - Return status: "created", "updated", or "unchanged"

**Core sync:**

5. `sync_patch(site, repo_root: Path, old_sha: str, new_sha: str, dry_run: bool = False) -> dict`
   - Call `build_patch_diff(old_sha, new_sha, repo_root)` to get PatchDiff
   - If PatchDiff has zero changes (no cards, no keywords, no enums), return `{"status": "no-changes", "version": diff.version}`
   - Generate wikitext via `patch_to_wikitext(diff)`
   - Page title: `Patch:{diff.version}`
   - Upsert the page (same idempotent pattern: compare rstrip, skip if unchanged)
   - Return `{"status": "created"/"updated"/"unchanged", "version": diff.version, "page": page_title}`

6. `sync_patch_index(site, repo_root: Path, dry_run: bool = False) -> str`
   - Scan all `Patch:*` pages on the wiki using `site.allpages(namespace=0, prefix="Patch:")` — but this won't work because Patch: is not a real namespace. Instead, use `site.categories["Patch"]` to find all pages in Category:Patch.
   - For each page, extract version from title (strip "Patch:" prefix), and read the page to extract date and commit from the Template:Patch invocation (regex: `\|date=(\S+)` and `\|commit=(\w+)`)
   - Build the patches list and call `patch_index_wikitext(patches)`
   - Upsert `Patch:Index` page
   - Return status

7. `sync_all_pending(site, repo_root: Path, dry_run: bool = False) -> list[dict]`
   - Load sync state to get last_synced_sha
   - Get current HEAD sha via `git rev-parse HEAD`
   - If last_synced_sha is None, use the very first commit as old_sha (or handle by doing a single diff from the parent of HEAD: `HEAD~1`)
   - If last_synced_sha == HEAD, return [] (nothing to sync)
   - Get list of commits between last_synced_sha and HEAD via `git rev-list --reverse {last_synced_sha}..HEAD`
   - For each commit in the list, check if it touches any of the watched paths (`data/cards/`, `data/GLOSSARY.md`, `src/grid_tactics/enums.py`) using `git diff-tree --no-commit-id -r --name-only {sha}`. If no watched files changed, skip.
   - For commits that DO touch watched files: call sync_patch with parent as old_sha and commit as new_sha (parent via `git rev-parse {sha}^`)
   - After processing all commits, update .sync_state.json with HEAD sha
   - Update Patch:Index
   - Return list of results from sync_patch calls

**CLI entry point:**

8. `main(argv=None) -> int`
   - argparse with:
     - `--pending`: sync all pending commits (default mode for hook)
     - `--commit SHA`: sync a specific commit (diff against its parent)
     - `--bootstrap-template`: just bootstrap Template:Patch
     - `--dry-run`: preview mode
   - Connect to wiki via get_site()
   - Run the appropriate function
   - Print results
   - `if __name__ == "__main__": raise SystemExit(main())`

**Important implementation notes:**
- All git subprocess calls use `cwd=repo_root`
- The watched paths for filtering commits: any file matching `data/cards/*.json`, `data/GLOSSARY.md`, `src/grid_tactics/enums.py`
- Handle edge case: first-ever sync (no .sync_state.json) by diffing HEAD against HEAD~1
- .sync_state.json should be gitignored (add to wiki/.gitignore if it exists, or note for user)

2. Add handler blocks (before the `# Connect to wiki` section):
   ```python
   if args.patch:
       from sync.sync_patches import sync_all_pending, bootstrap_patch_template
       try:
           site = get_site()
       except MissingCredentialsError as exc:
           print(f"ERROR: {exc}", file=sys.stderr)
           return 2
       # Ensure template exists
       tmpl_status = bootstrap_patch_template(site)
       if tmpl_status != "unchanged":
           print(f"  Template:Patch: {tmpl_status}")
       results = sync_all_pending(site, _REPO_ROOT, args.dry_run)
       if not results:
           print("No pending patch changes.")
       else:
           for r in results:
               print(f"  {r.get('page', r.get('version', '?'))}: {r['status']}")
       return 0
   ```

   Similar for `--patch-commit`.

**Part B: Create post-commit hook**

Create `.git/hooks/post-commit` (the file goes in the REPO ROOT's .git/hooks/, NOT wiki/.git/hooks/):

```bash
#!/bin/bash
# Grid Tactics Wiki — auto-sync patch notes on relevant file changes
#
# This hook runs after every commit. It checks if any watched files
# (card JSONs, GLOSSARY.md, enums.py) were modified. If so, it triggers
# the patch notes sync to update the wiki.

# Watched path patterns
WATCHED_PATTERNS="data/cards/|data/GLOSSARY.md|src/grid_tactics/enums.py"

# Check if this commit touches any watched files
CHANGED_FILES=$(git diff-tree --no-commit-id -r --name-only HEAD 2>/dev/null)
if ! echo "$CHANGED_FILES" | grep -qE "$WATCHED_PATTERNS"; then
    exit 0  # No relevant changes
fi

# Only run if wiki/.env exists (credentials available)
REPO_ROOT="$(git rev-parse --show-toplevel)"
if [ ! -f "$REPO_ROOT/wiki/.env" ]; then
    echo "[post-commit] wiki/.env not found, skipping patch sync"
    exit 0
fi

echo "[post-commit] Detected card/glossary/enum changes, syncing patch notes..."

# Run patch sync from the wiki directory
cd "$REPO_ROOT/wiki" || exit 0
python -m sync.sync_wiki --patch 2>&1 | sed 's/^/  [patch-sync] /'

# Don't fail the commit if sync fails (wiki might be unreachable)
exit 0
```

Make the hook executable. On Windows/Git Bash this means the file just needs to exist with the shebang line — git will execute it.
