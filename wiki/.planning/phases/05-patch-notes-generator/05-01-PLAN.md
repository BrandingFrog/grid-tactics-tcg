  <name>Task 1: Git-based diff engine (patch_diff.py)</name>
  <files>wiki/sync/patch_diff.py</files>
  <action>
Create `wiki/sync/patch_diff.py` with the following:

**Data structures (dataclasses):**
- `CardChange`: card_id, card_name, change_type (Literal["added", "changed", "removed"]), changed_fields (list[str], empty for added/removed)
- `KeywordChange`: keyword, change_type (Literal["added", "changed", "removed"]), old_description (str|None), new_description (str|None)
- `EnumChange`: enum_name (str, e.g. "EffectType"), value_name (str), change_type (Literal["added", "removed"])
- `PatchDiff`: version (str), cards (list[CardChange]), keywords (list[KeywordChange]), enums (list[EnumChange]), commit_sha (str), commit_date (str)

**Functions:**

1. `_git_show(sha: str, path: str, repo_root: Path) -> str | None`
   - Run `subprocess.run(["git", "show", f"{sha}:{path}"], capture_output=True, text=True, cwd=repo_root)`
   - Return stdout if returncode==0, else None (file didn't exist at that commit)

2. `_git_ls_tree(sha: str, dir_path: str, repo_root: Path) -> list[str]`
   - Run `git ls-tree --name-only {sha}:{dir_path}` to list files in a directory at a given commit
   - Return list of filenames (just basenames)

3. `diff_cards(old_sha: str, new_sha: str, repo_root: Path) -> list[CardChange]`
   - Use `_git_ls_tree` to get card filenames at old_sha and new_sha under `data/cards/`
   - For files only in new: added
   - For files only in old: removed
   - For files in both: parse JSON at both commits via `_git_show`, compare all fields. If any field differs, record as changed with the list of changed field names
   - Sort results by card_name

4. `diff_glossary(old_sha: str, new_sha: str, repo_root: Path) -> list[KeywordChange]`
   - Read `data/GLOSSARY.md` at both commits via `_git_show`
   - Parse the markdown table rows: extract keyword and description from each `| Keyword | Description |` row
   - Compare: added keywords, removed keywords, changed descriptions
   - Return list of KeywordChange

5. `diff_enums(old_sha: str, new_sha: str, repo_root: Path) -> list[EnumChange]`
   - Read `src/grid_tactics/enums.py` at both commits
   - Parse enum members using regex: find lines matching `^\s+([A-Z_]+)\s*=\s*\d+` within each `class XxxEnum(IntEnum):` block
   - Only track `EffectType` enum (the one relevant to card mechanics). Can expand later.
   - Compare members between commits: added, removed

6. `build_patch_diff(old_sha: str, new_sha: str, repo_root: Path) -> PatchDiff`
   - Read VERSION.json at new_sha via `_git_show` to get version string
   - Get commit date via `git log -1 --format=%Y-%m-%d {new_sha}`
   - Call diff_cards, diff_glossary, diff_enums
   - Return PatchDiff with all results
   - If PatchDiff has zero changes across all three categories, return it anyway (caller decides whether to skip)

**Important implementation notes:**
- All subprocess calls use `cwd=repo_root` so paths are relative to repo root
- Handle the case where GLOSSARY.md or enums.py didn't exist at old_sha (treat as empty)
- Card JSON comparison should ignore key ordering (parse both as dicts)
- Use `json.loads()` for card parsing, not string comparison

<task type="auto">
  <name>Task 2: Patch page wikitext generator (patch_page.py)</name>
  <files>wiki/sync/patch_page.py</files>
  <action>
Create `wiki/sync/patch_page.py` with the following:

**Functions:**

1. `patch_to_wikitext(diff: PatchDiff) -> str`
   Generate wikitext for a `Patch:X.Y.Z` page. Structure:

   ```
   {{Patch
   |version=X.Y.Z
   |date=YYYY-MM-DD
   |commit=<short sha>
   }}

   Patch '''X.Y.Z''' was released on YYYY-MM-DD.

   == Cards ==
   (only if diff.cards is non-empty)

   === Added ===
   (only if any added cards)
   * [[Card:Card Name|Card Name]] — new card
   (for each added card)

   === Changed ===
   (only if any changed cards)
   * [[Card:Card Name|Card Name]] — updated: field1, field2
   (for each changed card, listing changed_fields)

   === Removed ===
   (only if any removed cards)
   * [[Card:Card Name|Card Name]] — removed
   (for each removed card)

   == Mechanics ==
   (only if diff.keywords or diff.enums is non-empty)

   === Keywords ===
   (only if any keyword changes)
   * '''Keyword''' — added / changed / removed
   (for each keyword change; if changed, show new description)

   === Effect Types ===
   (only if any enum changes)
   * '''VALUE_NAME''' — added to EffectType / removed from EffectType

   [[Category:Patch]]
   ```

   - If a section would be empty, omit it entirely (no empty headings)
   - Card wikilinks use `[[Card:Display Name|Display Name]]` format
   - Changed fields list should be human-readable: replace underscores with spaces

2. `patch_index_wikitext(patches: list[dict]) -> str`
   Generate wikitext for `Patch:Index`. Input is a list of dicts with keys: version, date, commit_sha.
   Structure:

   ```
   This page lists all patches for Grid Tactics, newest first.

   {| class="wikitable sortable"
   |-
   ! Version !! Date !! Commit
   |-
   | [[Patch:X.Y.Z|X.Y.Z]] || YYYY-MM-DD || <code>abcdef1</code>
   |-
   | [[Patch:X.Y.Z|X.Y.Z]] || YYYY-MM-DD || <code>abcdef1</code>
   |}

   [[Category:Patch]]
   ```

   - Sort patches by version descending (newest first) using packaging.version or simple string sort
   - Short SHA (first 7 chars)

3. Also create a simple `Template:Patch` wikitext string as a module-level constant `PATCH_TEMPLATE_WIKITEXT`:
   ```
   <includeonly><div class="patch-infobox" style="float:right; border:1px solid #aaa; padding:8px; margin:0 0 8px 8px; width:250px; background:#f9f9f9;">
   '''Patch {{{version|}}}'''
   * '''Date:''' {{{date|}}}
   * '''Commit:''' <code>{{{commit|}}}</code>
   </div></includeonly><noinclude>
   Infobox template for patch notes pages.
   [[Category:Template]]
   </noinclude>
   ```

**Important:**
- Import PatchDiff from sync.patch_diff
- For version sorting, just use a simple reverse sort on version strings (they follow semver, lexicographic works for 0.x.y range)
- No wiki connection needed — pure wikitext string generation
