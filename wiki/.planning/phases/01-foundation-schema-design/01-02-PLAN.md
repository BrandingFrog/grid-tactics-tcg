Create `wiki/sync/__init__.py` (empty).

Create `wiki/sync/client.py` exporting `get_site()`:
- Load `.env` via `python-dotenv` from the wiki/ directory
- Read `MW_API_URL` (default `http://localhost:8080/`), `MW_BOT_USER`, `MW_BOT_PASS`
- Parse URL into host + path for mwclient.Site constructor (mwclient wants host without scheme, path with trailing slash)
- For localhost dev, pass `scheme='http'` to Site; for prod this will be `https`
- Return authenticated `mwclient.Site` instance. Raise clear error if creds missing.

Use `mwclient.Site(host, path='/w/', scheme='http')` — Taqasta serves MW at `/w/` with short URLs rewriting `/wiki/X` to `index.php?title=X`. **Confirm the exact path by curling `http://localhost:8080/w/api.php?action=query&meta=siteinfo&format=json`** before locking the path; if it 404s, try `/`.

Add new vars to `wiki/.env.example`:
```

MW_BOT_USER=SyncBot@phase1
MW_BOT_PASS=
```

Install deps: from `wiki/` run `pip install -e .` (or `uv pip install -e .` if uv is available). Confirm `python -c "import mwclient; print(mwclient.__version__)"` works.

<task type="auto">
  <name>Task 2: Create BotPassword and verify SMW is installed</name>
  <files>wiki/sync/verify_smw.py, wiki/.env</files>
  <action>
Taqasta auto-installs SMW on first boot if `SemanticMediaWiki` is in `MW_LOAD_EXTENSIONS` (set in Plan 01-01). This task verifies that and creates the bot credential.

**Step A — Confirm SMW loaded:**
Curl the API for extension list:
```
curl -s "http://localhost:8080/w/api.php?action=query&meta=siteinfo&siprop=extensions&format=json" | python -m json.tool
```
Grep for `"SemanticMediaWiki"` in the output. If missing: `docker compose logs mediawiki | grep -i semantic` to diagnose, then fix `MW_LOAD_EXTENSIONS` in .env, `docker compose restart mediawiki`, wait 60s, retry.

**Step B — Create BotPassword via web UI:**
This step requires the browser (MediaWiki does not expose BotPassword creation via API without an existing session). Use Playwright MCP or `curl` with login cookies:

1. Log in as admin via Special:UserLogin (POST with lgname=admin, lgpassword from .env)
2. Navigate to `Special:BotPasswords`
3. Create a bot with name `phase1`, grants: `High-volume editing`, `Edit existing pages`, `Create, edit, and move pages`
4. Capture the generated password (shown once)

If Playwright MCP is not available in execution, fall back to MediaWiki maintenance script inside the container:
```
docker compose exec mediawiki php maintenance/createBotPassword.php \
  --user admin --appid phase1 \
  --grants highvolume,editpage,createeditmovepage
```
(Taqasta ships maintenance scripts at `/var/www/mediawiki/maintenance/` — confirm path with `docker compose exec mediawiki find / -name createBotPassword.php 2>/dev/null | head -1`.)

If no maintenance script exists, use mwclient-on-admin-session approach: script logs in as admin with password auth, POSTs to `Special:BotPasswords` form. Prefer the maintenance script path if available — cleaner.

Save the resulting bot password into `wiki/.env` as `MW_BOT_PASS=<value>`. The full credential mwclient uses is `MW_BOT_USER=SyncBot@phase1` (admin's bot, named phase1) — update .env accordingly, and adjust if you used a different username.

**Step C — Write verify_smw.py:**
Create `wiki/sync/verify_smw.py` — a CLI that:
1. Calls `get_site()` from client.py
2. Calls `site.api('query', meta='siteinfo', siprop='extensions')` and asserts "SemanticMediaWiki" in returned extensions
3. Prints `SMW version: <version>` and `Bot authenticated as: <username>` (from `site.username`)
4. Runs a trivial `site.ask("[[Modification date::+]]|limit=1")` to prove the ask API works end-to-end
5. Exits 0 on success, non-zero with clear message on failure

Make it runnable as `python -m sync.verify_smw` from `wiki/`.
  </action>
  <verify>
`cd wiki && python -m sync.verify_smw` exits 0 and prints SMW version + authenticated username + ask() result count.
  </verify>
  <done>SMW confirmed installed via API, BotPassword created, credentials in .env, verify_smw.py passes.</done>
</task>
