<task type="auto">
  <name>Task 1: Create docker-compose.yml with 3 services</name>
  <files>wiki/docker-compose.yml, wiki/.env.example, wiki/.gitignore</files>
  <action>
Create `wiki/docker-compose.yml` using the exact topology from RESEARCH.md section 4 ("Local dev compose file"). Three services:

1. **mediawiki** — `image: ghcr.io/wikiteq/taqasta:latest`, ports `8080:80`, depends_on `db`, volume `mw_data:/mediawiki`. Env vars sourced from `.env` via `env_file: .env`:
   - MW_SITE_NAME, MW_SITE_SERVER=http://localhost:8080, MW_SITE_LANG=en
   - MW_ADMIN_USER, MW_ADMIN_PASS
   - MW_DB_TYPE=mysql, MW_DB_SERVER=db, MW_DB_NAME, MW_DB_USER, MW_DB_PASS
   - MW_MAIN_CACHE=CACHE_REDIS, MW_REDIS_SERVERS=redis:6379
   - MW_LOAD_EXTENSIONS=SemanticMediaWiki,SemanticResultFormats,PageForms,ParserFunctions,Arrays,Scribunto,CategoryTree
   - MW_ENABLE_UPLOADS=true, MW_FILE_EXTENSIONS=png,jpg,jpeg,webp,svg
   - MW_SECRET_KEY

2. **db** — `image: mariadb:10.11`, volume `db_data:/var/lib/mysql`, env vars MARIADB_ROOT_PASSWORD, MARIADB_DATABASE, MARIADB_USER, MARIADB_PASSWORD from `.env`.

3. **redis** — `image: redis:7-alpine`, volume `redis_data:/data`.

Named volumes at bottom: `mw_data`, `db_data`, `redis_data`.

Then create `wiki/.env.example` with dev dummy values for every env var referenced above (e.g., `MW_ADMIN_PASS=devpassword`, `MARIADB_ROOT_PASSWORD=rootpass`, `MW_SECRET_KEY=dev-secret-change-me-32chars-min`). Include header comment: "Copy to .env and adjust. DO NOT commit .env."

Create or append `wiki/.gitignore` with: `.env`, `*.log`, `__pycache__/`.

**Critical:** Do NOT use `version: "3.9"` top key (deprecated in Compose v2). Use `ghcr.io/wikiteq/taqasta:latest` exactly — this is the image Railway's official template uses (per RESEARCH.md §2), switching images later breaks Phase 2.
  </action>
  <verify>
`docker compose -f wiki/docker-compose.yml config` exits 0 and prints resolved config with all 3 services.
`test -f wiki/.env.example && test -f wiki/.gitignore` both pass.
`grep -c "ghcr.io/wikiteq/taqasta" wiki/docker-compose.yml` returns 1.
  </verify>
  <done>Compose file validates, all 3 services defined, env vars externalized to .env.example, secrets git-ignored.</done>
</task>

<task type="auto">
  <name>Task 3: Boot smoke test</name>
  <files>wiki/docker-compose.yml</files>
  <action>
From `wiki/` directory: run `cp .env.example .env` (if .env doesn't exist), then `docker compose up -d`. Wait up to 120 seconds for mediawiki to become healthy (poll with `curl -s -o /dev/null -w "%{http_code}" http://localhost:8080`). Expect HTTP 200 or 301/302 (Taqasta may redirect to Main_Page).

If boot fails: `docker compose logs mediawiki` — common causes: MW_SECRET_KEY too short, DB not ready (Taqasta retries automatically, give it 90s), port 8080 conflict.

Once HTTP 200 confirmed: run `docker compose ps` to verify all 3 services "Up". Leave the stack running for Plan 01-02 to use.

**Do not** proceed to verify until `curl http://localhost:8080` returns MediaWiki HTML (look for `<title>` containing "Grid Tactics" or "Main Page").
