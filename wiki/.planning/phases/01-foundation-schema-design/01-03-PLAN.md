Define a helper `property_wikitext(name, smw_type, description, allowed_values=None) -> str` that builds the wikitext:
```
This property stores <description>.

[[Has type::<Type>]]
{optional [[Allows value::X]] lines}
```

<task type="auto">
  <name>Task 2: Write and run bootstrap_schema.py + verify_schema.py</name>
  <files>wiki/sync/bootstrap_schema.py, wiki/sync/verify_schema.py</files>
  <action>
**bootstrap_schema.py:**
- Imports `get_site` from client.py, `PROPERTIES` + `EFFECT_SUBPROPERTIES` + `property_wikitext` from schema.py
- For each property in PROPERTIES + EFFECT_SUBPROPERTIES:
  - `page = site.pages[f"Property:{name}"]`
  - Build expected wikitext via helper
  - If `page.text() == expected`: skip (log "unchanged: Property:X")
  - Else: `page.edit(expected, summary="bootstrap schema from wiki/sync/schema.py")` (log "updated: Property:X")
- Print summary: N created, M updated, K unchanged
- Must be **idempotent** — second run prints K=all, N=M=0

**verify_schema.py:**
- Imports `get_site` and `PROPERTIES` + `EFFECT_SUBPROPERTIES`
- For each expected property:
  - Fetch `Property:<Name>` page text
  - Assert page exists and contains `[[Has type::<ExpectedType>]]`
- Query `site.ask("[[Property:+]]|?Has type|limit=500")` as a cross-check (lists all defined properties with their types)
- Build a report: {name: declared_type_from_page} vs {name: expected_type_from_schema}
- Print table of mismatches if any, exit 1 on mismatch, exit 0 on full match
- Print total count on success: "Schema OK: N/N properties defined correctly"

**Execute both:**
1. `cd wiki && python -m sync.bootstrap_schema` — expect ~25 "created" messages on first run
2. `python -m sync.bootstrap_schema` again — expect "unchanged" for all
3. `python -m sync.verify_schema` — expect "Schema OK: 25/25"

**Edge case to handle:** mwclient's `page.text()` may return empty string for non-existent pages. Use `page.exists` attribute to distinguish "doesn't exist" from "empty" — create if missing, update if different text.
  </action>
  <verify>
Run sequence above. Second bootstrap run must show zero updates. verify_schema exits 0.
Manual spot check: curl `http://localhost:8080/wiki/Special:Properties` and confirm CardType, Element, Cost, HasEffect all appear with correct types.
  </verify>
  <done>All properties live on wiki, bootstrap is idempotent, verify passes.</done>
