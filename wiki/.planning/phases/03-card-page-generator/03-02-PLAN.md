**Step 1: Fix Category name in Template:Card**

Edit `wiki/sync/templates/Card.wiki` line 66: change `[[Category:Cards]]` to `[[Category:Card]]`.

This is the ONLY change needed in the template. Do NOT change the `[[Category:{{{type}}}s]]` or `[[Category:{{{element}}} cards]]` lines — those are fine for subcategories.

After editing the local file, re-run `bootstrap_template.py` to push the update to the live wiki:
```bash
cd wiki && python -m sync.bootstrap_template
```

**Step 2: Verify file upload capability**

Write a small test script (can be inline Python or a temporary script) that:
1. Connects to the wiki via `get_site()`
2. Creates a tiny 1x1 PNG in memory (use Python's struct or a hardcoded minimal PNG bytestring)
3. Attempts `site.upload(file=..., filename="Test_upload.png", description="Upload test", ignore=True)`
4. If it succeeds, delete the test file page afterward
5. If it fails with permission error, report the exact error

If upload fails due to `$wgEnableUploads = false` or missing bot rights:
- Document the error in the SUMMARY
- This becomes a blocker for Plan 03 (art upload) but NOT for card page creation

**Step 3: Upload CardBack.png placeholder**

Create a simple card-back placeholder image. Options (in order of preference):
1. Check if there's already a card back image anywhere in the repo (e.g., `src/grid_tactics/server/static/` or `web-dashboard/`)
2. If not, generate a minimal 280x400 dark gray PNG with Python (use PIL/Pillow if available, or create a minimal valid PNG with raw bytes — a solid color rectangle is fine)
3. Upload it to the wiki as `CardBack.png` using `site.upload()`

The placeholder just needs to not be a broken image. A dark rectangle matching the card template's `#1a1a1a` background is ideal.

**Step 4: Verify the fix**

After pushing the template update:
1. Run `python -m sync.create_sample_card` to re-sync Card:Ratchanter with the updated template

3. Check that CardBack.png exists on the wiki: `site.images["CardBack.png"].exists`
