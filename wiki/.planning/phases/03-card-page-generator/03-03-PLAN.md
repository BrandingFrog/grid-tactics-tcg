<task type="auto">
  <name>Task 1: Build sync_wiki.py CLI with card upsert + art upload</name>
  <files>wiki/sync/sync_wiki.py</files>
  <action>
Create `wiki/sync/sync_wiki.py` with argparse CLI and the following subcommands/flags:

**CLI interface:**
```
python -m sync.sync_wiki --all-cards         # Upsert all 34 cards + upload art
python -m sync.sync_wiki --card ratchanter   # Upsert single card by card_id
python -m sync.sync_wiki --all-cards --dry-run  # Show what would change
python -m sync.sync_wiki --verify            # Check Category:Card count matches JSON count
python -m sync.sync_wiki --upload-art        # Upload all art PNGs only (no page edits)
```

**Core functions:**

1. **`load_all_cards(cards_dir: Path) -> list[dict]`**
   - `cards_dir.glob("*.json")` -> load each with `json.load()`
   - Sort by card_id for deterministic output
   - Return list of raw card dicts

2. **`upsert_card_page(site, card: dict, name_map: dict, art_dir: Path, dry_run: bool) -> dict`**
   - Determine art_exists: check if `art_dir / f"{card['card_id']}.png"` exists
   - Generate wikitext via `card_to_wikitext(card, name_map, art_exists=art_exists)`
   - Page title: `f"Card:{card['name']}"`
   - Compare with existing page text (rstrip-normalized, same pattern as create_sample_card.py)
   - If dry_run: return status without editing
   - Return dict with `{"card_id": ..., "page": ..., "status": "created|updated|unchanged"}`
   - Use edit summary: `f"sync {card['name']} from data/cards/{card_id}.json"`

3. **`upload_card_art(site, card_id: str, card_name: str, art_dir: Path, dry_run: bool) -> str`**
   - Check if `art_dir / f"{card_id}.png"` exists; if not, return "no-art"
   - If dry_run, return "dry-run"
   - Upload using `site.upload(file=f, filename=f"{card_id}.png", description=f"Card art for [[Card:{card_name}]].", ignore=True, comment="sync card art")`
   - Return "uploaded" on success, "error" on exception (print the error)

4. **`verify_card_count(site, expected: int) -> bool`**
   - Count members of `site.categories["Card"]` using `sum(1 for _ in cat)`
   - Print actual vs expected
   - Return True if match

5. **`main()`**
   - Parse args
   - Paths: `cards_dir = Path(__file__).resolve().parent.parent.parent / "data" / "cards"` and `art_dir = Path(__file__).resolve().parent.parent.parent / "src" / "grid_tactics" / "server" / "static" / "art"`
   - Build name_map via `build_card_name_map(cards_dir)`
   - Connect to wiki via `get_site()`
   - For `--all-cards`: iterate all cards, upsert page + upload art for each. Print summary table at end (created/updated/unchanged/art counts).
   - For `--card <id>`: load single card JSON, upsert + upload art
   - For `--verify`: run `verify_card_count()` with expected = number of JSON files
   - For `--upload-art`: upload art only (no page edits)
   - For `--dry-run`: works with any of the above, suppresses actual edits/uploads
   - Exit code: 0 on success, 1 on verification failure, 2 on connection error

**Idempotency:** The `rstrip()` comparison pattern from `create_sample_card.py` ensures re-running with unchanged data produces zero edits.

**Error handling:**
- Catch `MissingCredentialsError` and print helpful message
- Catch individual card errors (log and continue to next card, don't abort batch)
- Print a final summary: `Synced 34 cards: 34 created, 0 updated, 0 unchanged. Art: 15 uploaded, 19 no-art.`

**Important:** The script must be runnable as `cd wiki && python -m sync.sync_wiki --all-cards` (module execution from wiki/ directory).
