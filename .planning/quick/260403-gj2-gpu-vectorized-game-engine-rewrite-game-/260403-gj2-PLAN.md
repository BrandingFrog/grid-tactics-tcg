Game flow per step:
1. ACTION phase: active player picks one of 7 action types
2. Action resolved (deploy minion / cast magic / move / attack / sacrifice / draw / pass)
3. Dead minion cleanup + on_death effects
4. Win check (player HP <= 0)
5. Transition to REACT phase (opponent can counter)
6. REACT phase: react player plays PLAY_REACT or PASS
7. If PLAY_REACT: push to stack, swap react player (counter-react chain)
8. If PASS: resolve stack LIFO (NEGATE cancels next entry), cleanup, win check
9. Advance turn: flip active player, increment turn, regen mana

**CRITICAL DESIGN RULES (from research):**
- ALL tensors have leading batch dimension N. No Python loops over the batch dimension.
- ALL variable-length structures use fixed-size tensors with padding + size counters (Pitfall 3).
- ALL conditionals use `torch.where()` / masked operations, NOT Python if/else (Pitfall 1).
- ALL game logic stays on GPU. Only transfer to CPU at VecEnv boundary (Pitfall 2).
- Use in-place operations (`.masked_fill_()`, `[mask] = value`) where safe to save VRAM (Pitfall 5).
- Sentinel value -1 for empty slots (card IDs, minion slots, board cells).

**effects.py** -- Batched effect resolution via CardTable lookup:
- `apply_effects_batch(state, card_ids, trigger, caster_owners, caster_slots, target_flat_pos, card_table) -> state`
  - Loops over `range(MAX_EFFECTS_PER_CARD)` (fixed 3 iterations, not Python-over-batch).
  - For each effect index, gathers effect properties from card_table using card_ids.
  - Builds `active` mask: trigger matches AND effect exists (effect_idx < num_effects).
  - Dispatches to effect handlers by type, all computed and merged via masks:
    - `_apply_damage_batch()`: SINGLE_TARGET -> find minion at target_flat_pos, reduce health. ALL_ENEMIES -> mask all enemy minions, reduce health. ADJACENT -> mask minions adjacent to caster. SELF_OWNER -> reduce caster's owner player HP.
    - `_apply_heal_batch()`: Like damage but add health, capped at card_table.health for minions or STARTING_HP for players.
    - `_apply_buff_attack_batch()`: Add to minion_atk_bonus for SELF_OWNER target.
    - `_apply_buff_health_batch()`: Add to minion_health for SELF_OWNER target (no cap).
  - NEGATE effects are no-ops here (handled in react resolution).
  - DEPLOY_SELF is handled separately in react resolution.
  - For ADJACENT target: use the precomputed adjacency tensor from CardTable. Given caster position flat index, look up the 8 adjacent cells (orthogonal + diagonal per the Python engine's `Board.get_all_adjacent`), find minions at those positions, apply effect.
  - For SELF_OWNER: DAMAGE/HEAL target the owning player HP. BUFF_ATTACK/BUFF_HEALTH target the caster's own minion.
  - For finding "minion at position": build board-to-slot lookup from `state.board` tensor. `board[n, row, col]` gives the minion slot index, then index into minion arrays.

Then dispatch ALL action types simultaneously (compute all results, merge via action_type mask):

- `apply_pass_batch(state, mask)`: No-op on state. (React transition happens in engine.)

- `apply_draw_batch(state, mask, card_table)`:
  - For each masked game: read `decks[n, active_player, deck_tops[n, active_player]]` to get card_id.
  - Add to hand at `hand_sizes[n, active_player]` position.
  - Increment `deck_tops` and `hand_sizes`.
  - Use `torch.where(mask, new_val, old_val)` for each field.

- `apply_move_batch(state, mask)`:
  - Source position from action decode, direction from action decode.
  - Compute destination = source + direction delta (use precomputed [4,2] delta tensor).
  - Find minion slot at source via `state.board[n, src_row, src_col]`.
  - Clear board at source, set board at dest to minion slot.
  - Update `minion_row`, `minion_col` for the moved minion.

- `apply_play_card_batch(state, mask, card_table)`:
  - Get card from hand: `card_id = state.hands[n, active_player, hand_idx]`.
  - Check card_type via card_table to split into minion deploy vs magic.
  - **Minion deploy**: allocate next minion slot, set minion fields (card_id, owner, row, col, health from card_table, alive=True), place on board, remove card from hand, add to graveyard, spend mana.
  - **Magic cast**: remove card from hand, add to graveyard, spend mana, then call `apply_effects_batch` with ON_PLAY trigger.
  - Use `is_minion = card_table.card_type[card_id] == 0` mask to split paths, compute both, merge.

- `apply_attack_batch(state, mask, card_table)`:
  - Source (attacker) and target (defender) from action decode as flat positions.
  - Find minion slots via board lookup.
  - Look up base attack from card_table for both attacker and defender.
  - Compute effective attack = card_table.attack[card_id] + minion_atk_bonus.
  - Apply simultaneous damage: attacker_health -= defender_effective, defender_health -= attacker_effective.
  - Then trigger ON_ATTACK effects for attacker, ON_DAMAGED for both (if damaged).
  - Use `apply_effects_batch` for each trigger.

- `apply_sacrifice_batch(state, mask, card_table)`:
  - Find minion at source position.
  - Compute effective_attack = card_table.attack + minion_atk_bonus.
  - Remove minion from board (set board cell to -1, set minion_alive=False).
  - Add card to owner's graveyard.
  - Deal effective_attack damage to opponent player HP.

- `apply_react_batch(state, mask, card_table)`:
  - Get card from react_player's hand.
  - Determine cost: react_mana_cost for multi-purpose, mana_cost for REACT cards.
  - Spend mana, remove from hand, add to graveyard.
  - Push entry onto react_stack at react_stack_depth position.
  - Increment react_stack_depth.
  - Swap react_player to other player.

Each function operates on the full batch but only modifies games where `mask` is True.

**Hand management helpers** (in actions.py or state.py):
- `_remove_from_hand(state, player_idx_per_game, hand_idx)`: Shift cards left to fill gap, decrement hand_sizes.
- `_add_to_graveyard(state, player_idx_per_game, card_id)`: Place at graveyard_sizes position, increment.
- `_spend_mana(state, player_idx_per_game, cost)`: Subtract from player_mana.

**react.py** -- Batched react system:
- `resolve_react_stack_batch(state, card_table) -> state`:
  - Iterates `for depth in range(MAX_REACT_DEPTH - 1, -1, -1)` (LIFO order, 10 fixed iterations).
  - Builds `active` mask: games where `react_stack_depth > depth`.
  - Tracks `negated` mask: entries negated by prior NEGATE resolution.
  - For each active, non-negated entry:
    - Look up card from react_stack entry's card_numeric_id.
    - If REACT card: check for NEGATE effects -> mark next entry as negated. Resolve non-NEGATE ON_PLAY effects via apply_effects_batch.
    - If multi-purpose: check react_effect. DEPLOY_SELF -> allocate minion slot, place on board. Other -> resolve via apply_effects_batch.
  - After full stack resolution:
    - Call `cleanup_dead_minions_batch` (see engine.py).
    - Call `check_game_over_batch`.
    - For non-game-over games: clear react state, flip active_player, increment turn_number, regen mana.

        """Apply one action per game. Full game loop:

        3. If REACT phase: apply PASS (resolve stack) or PLAY_REACT (push to stack)
        """

        """Remove minions with health <= 0. Trigger ON_DEATH effects. Add to graveyard."""

    def check_game_over_batch(self) -> None:

**legal_actions.py** -- `compute_legal_mask_batch(state, card_table) -> [N, 1262] bool`:

This is the most complex single function. It must match the Python engine's `legal_actions()` exactly. Compute the entire [N, 1262] mask as tensor operations.

For ACTION phase games (`state.phase == 0`):

- **PLAY_CARD [0:250]** -- For each hand slot (loop `range(MAX_HAND=10)`):
  - `card_id = state.hands[arange_N, active_p, hand_idx]` -- [N]
  - `has_card = card_id >= 0` -- [N]
  - `can_afford = state.player_mana[arange_N, active_p] >= card_table.mana_cost[card_id.clamp(min=0)]` -- [N]
  - `is_minion = card_table.card_type[card_id.clamp(min=0)] == 0` -- [N]
  - `is_magic = card_table.card_type[card_id.clamp(min=0)] == 1` -- [N]
  - `is_react_type = card_table.card_type[card_id.clamp(min=0)] == 2` -- [N]
  - Skip react cards in ACTION phase: `playable = has_card & can_afford & ~is_react_type`
  - For minions: compute valid deploy positions per deployment rules:
    - P1 (active_p==0): melee (range==0) -> rows 0,1; ranged (range>=1) -> row 0 only
    - P2 (active_p==1): melee -> rows 3,4; ranged -> row 4 only
    - Cell must be empty: `state.board[n, row, col] == -1`
    - Build [N, 25] deploy mask, then set `mask[:, PLAY_CARD_BASE + hand_idx * 25 : PLAY_CARD_BASE + hand_idx * 25 + 25]`
  - For magic: check if any ON_PLAY effect has SINGLE_TARGET. If yes, enumerate enemy minion positions as valid targets. If no, only cell 0 is valid (untargeted).
  - NOTE on minion ON_PLAY SINGLE_TARGET: Current Python engine allows deploying with target_pos pointing at enemy minions. For simplicity in V1, allow all deploy positions without target enumeration (the tensor engine can apply effects to nearest enemy or skip if no target). This simplification is acceptable because the legal action mask already constrains deploy positions -- the target dimension in the PLAY_CARD encoding actually encodes EITHER deploy position OR magic target, not both. Per the action_space.py encoding: `PLAY_CARD_BASE + hand_idx * 25 + cell` where cell = deploy position for minions, target position for targeted magic. So for minions, enumerate valid deploy cells. For targeted magic, enumerate cells with enemy minions.

- **MOVE [250:350]** -- For each board position (loop `range(25)` or vectorize):
  - `minion_slot = state.board[n, row, col]` -- find occupied cells with owned minions.
  - For each occupied cell owned by active player, check 4 orthogonal neighbors.
  - Neighbor valid if in bounds AND `state.board[n, neighbor_row, neighbor_col] == -1`.
  - Set `mask[:, MOVE_BASE + src_flat * 4 + direction]`.
  - Can vectorize: build [N, 25] owned_minion mask, then for each of 4 directions, compute neighbor validity.

- **SACRIFICE [975:1000]** -- Owned minions on opponent's back row:

  Actually, reviewing the Python code: ACTION phase has no PASS in `_action_phase_actions`. PASS is only in REACT phase. If no actions available, empty tuple = auto-lose. So `mask[:, PASS_IDX] = False` for ACTION phase. But the env.py wraps with fallback-to-PASS for compatibility.   - Card must be react-eligible (CardType.REACT or is_multi_purpose in card_table).

def _compare_state(py_state, tensor_state, game_idx, step_num, lib):
    """Compare Python GameState with tensor engine state for one game."""
    prefix = f"Step {step_num}, game {game_idx}"
    
    # Player HP
    for p in range(2):
        py_hp = py_state.players[p].hp
        t_hp = tensor_state.player_hp[game_idx, p].item()
        assert py_hp == t_hp, f"{prefix}: P{p} HP mismatch: py={py_hp}, tensor={t_hp}"
    
    # Player mana
    for p in range(2):
        py_mana = py_state.players[p].current_mana
        t_mana = tensor_state.player_mana[game_idx, p].item()
        assert py_mana == t_mana, f"{prefix}: P{p} mana mismatch: py={py_mana}, tensor={t_mana}"

    def test_random_games_match(self, setup):
        """Run 8 random games for 50 steps each, comparing state at every step."""
        lib, ct, deck_tuple, deck_tensor, device = setup
        N = 8
        MAX_STEPS = 50
        encoder = ActionEncoder()
        
        # Initialize both engines with same seed
        seed = 42
        
        # Python engine: N separate games
        py_states = []
        py_rngs = []
        for i in range(N):
            state, rng = GameState.new_game(seed + i, deck_tuple, deck_tuple)
            py_states.append(state)
            py_rngs.append(rng)
        
        # Tensor engine: one batched engine
        engine = TensorGameEngine(
            N, ct, deck_tensor.expand(N, -1), deck_tensor.expand(N, -1), device,
            seeds=torch.arange(seed, seed + N, device=device),
        )
        engine.reset_batch()
        
        # Compare initial state
        for i in range(N):
            _compare_state(py_states[i], engine.state, i, 0, lib)
        
        rng = np.random.RandomState(12345)
        
        for step in range(MAX_STEPS):
            # For each game, pick a random legal action using Python engine
            action_ints = []
            py_actions = []
            
            for i in range(N):
                if py_states[i].is_game_over:
                    action_ints.append(PASS_IDX)
                    py_actions.append(None)
                    continue
                
                # Get legal actions from Python engine
                py_legal = legal_actions(py_states[i], lib)
                if not py_legal:
                    action_ints.append(PASS_IDX)
                    py_actions.append(None)
                    continue
                
                # Pick random legal action
                chosen = py_legal[rng.randint(len(py_legal))]
                py_actions.append(chosen)
                
                # Encode to integer
                action_int = encoder.encode(chosen, py_states[i])
                action_ints.append(action_int)
            
            # Step Python engine
            for i in range(N):
                if py_actions[i] is not None and not py_states[i].is_game_over:
                    py_states[i] = resolve_action(py_states[i], py_actions[i], lib)
            
            # Step tensor engine
            action_tensor = torch.tensor(action_ints, device=device, dtype=torch.int64)
            engine.step_batch(action_tensor)
            
            # Compare states
            for i in range(N):
                if not py_states[i].is_game_over:
                    _compare_state(py_states[i], engine.state, i, step + 1, lib)
