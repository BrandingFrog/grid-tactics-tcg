Implement PLAY-02: React window UI. Show the reacting player what the opponent is about to do, highlight react cards in hand, and let them either play a react card or skip (skip is already wired).

Purpose: Without this UI, the react phase is invisible — the player sees their phase badge change to "REACT" but has no idea what they are reacting to and no visual prompt that their react cards are now playable. **Step A — game.js: helper to detect react window and react legal cards.**

Add two helpers near `getLegalByType` (around line 1214):

```js
// True when the server has put us in a react window
function isReactWindow() {
    return gameState
        && gameState.phase === 1
        && gameState.react_player_idx === myPlayerIdx
        && legalActions
        && legalActions.length > 0;
}

// Hand indices that are legal as PLAY_REACT (action_type 5)
function getLegalReactCardIndices() {
    var indices = {};
    legalActions.forEach(function(a) {
        if (a.action_type === 5 && a.card_index != null) {
            indices[a.card_index] = true;
        }
    });
    return indices;
}
```

**Step B — game.js: pending action description.**

Add `describePendingAction(pa)` immediately after `getLegalReactCardIndices`. It must read `gameState.pending_action` (shape in <interfaces>) and return a short human string. Use `cardDefs[numeric_id].name` for card names and `getMinionAt(row, col)` to look up attacked minions. Cover these cases (action_type values in interfaces block):

- `action_type === 0` (PLAY_CARD): If `pa.position` exists, "Opponent is playing a card at row {row+1} col {col+1}". If no position (magic), "Opponent is casting a card". - `action_type === 2` (ATTACK): use `pa.target_id` to find minion via `gameState.minions.find(function(m){return m.minion_id === pa.target_id;})`. If not found, "Opponent is attacking".
- `action_type === 1` (MOVE): "Opponent is moving a minion".
- `action_type === 6` (SACRIFICE): minion = `gameState.minions.find(function(m){return m.minion_id === pa.minion_id;})`; "Opponent is sacrificing their {name} for damage" (or just "Opponent is sacrificing for damage" if not found).

if (isReactWindow()) {

**Step E — game.js: route hand-card clicks during react.**

At the very top of `onHandCardClick(handIdx)` (line 1279), BEFORE the existing `isMyTurn` check, insert:

```js
// Phase 14 PLAY-02: react window has its own click semantics
if (isReactWindow()) {
    var reactAction = legalActions.find(function(a) {
        return a.action_type === 5 && a.card_index === handIdx;
    });
    if (reactAction) {
        var payload = { action_type: 5, card_index: handIdx };
        if (reactAction.target_pos) payload.target_pos = reactAction.target_pos;
        submitAction(payload);
    }
    return;  // do not fall through to ACTION-phase logic
}
```
