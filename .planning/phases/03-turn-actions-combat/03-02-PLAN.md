    Create src/grid_tactics/action_resolver.py with the main entry point and per-action-type handlers:

    1. **`resolve_action(state: GameState, action: Action, library: CardLibrary) -> GameState`**: Main entry point. Validates `state.phase == TurnPhase.ACTION`. Dispatches to handler by `action.action_type`. After the action handler returns, calls `_cleanup_dead_minions(state, library)` per D-02, then transitions to react window: `replace(state, phase=TurnPhase.REACT, react_player_idx=1-state.active_player_idx, pending_action=action)`. For PASS, still open react window (per D-13: react window opens after every action).

    2. **`_apply_pass(state: GameState) -> GameState`**: Return state unchanged. (The react transition happens in resolve_action.)

    3. **`_apply_draw(state: GameState) -> GameState`**: Call `player.draw_card()` on the active player. Replace player in state. If deck is empty, raise ValueError (legal_actions in Plan 03 will prevent this).

    4. **`_apply_move(state: GameState, action: Action) -> GameState`**: Find minion by `action.minion_id` in state.minions. Validate minion exists and belongs to active player. Validate `action.position` is orthogonally adjacent to minion's current position. Validate target cell is empty. Update: Board.remove(old_pos) then Board.place(new_pos), replace minion with new position. 

    5. **`_apply_play_card(state: GameState, action: Action, library: CardLibrary) -> GameState`**: Get `card_numeric_id = player.hand[action.card_index]`. Look up card_def. If `card_def.card_type == CardType.MINION`: validate deployment zone per D-08/D-09 (melee: any friendly row; ranged: back row only). Spend mana, discard from hand, create MinionInstance, place on board, increment next_minion_id. Then trigger ON_PLAY effects via `resolve_effects_for_trigger`. If `card_def.card_type == CardType.MAGIC`: spend mana, discard from hand, resolve all ON_PLAY effects (effects may need target_pos from action.target_pos per D-10). React cards cannot be played during ACTION phase (raise ValueError).

    6. **`_apply_attack(state: GameState, action: Action, library: CardLibrary) -> GameState`**: Find attacker and defender minions. Validate attacker belongs to active player, defender does NOT belong to active player. Validate attack range with `_can_attack(attacker, defender, attacker_card_def)`:
       - `attack_range == 0` (melee): `manhattan_distance == 1` AND same row or same column (orthogonal only)
       - `attack_range >= 1` (ranged): (orthogonal AND `manhattan_distance <= attack_range`) OR (not orthogonal AND `chebyshev_distance == 1`, i.e., diagonal adjacent)
       Apply simultaneous damage per D-01: `new_attacker.current_health -= defender_card.attack`, `new_defender.current_health -= attacker_card.attack`. Use `attacker_card.attack + attacker.attack_bonus` as effective attack for the attacker, and `defender_card.attack + defender.attack_bonus` for the defender. Trigger ON_ATTACK effects for attacker, ON_DAMAGED for both.

    7. **`_cleanup_dead_minions(state: GameState, library: CardLibrary) -> GameState`**: Per D-02: find all minions with `is_alive == False`. Remove them from board (Board.remove) and from minions tuple. Collect on_death effects sorted by instance_id (ascending, per research discretion decision). Resolve on_death effects on the post-removal state. Cards of dead minions move to their owner's graveyard via `player.discard_from_hand` -- no, actually the card was already discarded from hand when deployed; dead minions are just removed. The card_numeric_id should go to the owner's graveyard. Add card_numeric_id to owner player's graveyard tuple.

    8. **Helper `_replace_player(players: tuple[Player, Player], idx: int, new_player: Player) -> tuple[Player, Player]`**: Returns new players tuple with one player replaced.
