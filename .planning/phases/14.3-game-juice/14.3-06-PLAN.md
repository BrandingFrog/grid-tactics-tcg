<semantics_decision>
**Burn tick semantics (locked):**
- **Who ticks:** ALL minions on the board with `burning_stacks > 0`, regardless of owner.
- **When ticks:** Once per turn flip, inside `react_stack.resolve_react_stack`, AFTER the react resolves and the active player has flipped, BEFORE mana regen for the new active player. This guarantees a burning minion takes exactly one tick per full turn cycle and that the new active player sees post-tick state when they begin their turn.
- **Damage formula:** `damage = burning_stacks * BURN_DPT` where `BURN_DPT = 1` (constant defined in `minion.py` or a new `constants.py` near MinionInstance — planner picks).
- **Decrement:** After dealing damage, `burning_stacks -= 1` (clamped >= 0).
- **Death:** If `current_health <= 0` after burn damage, route through the existing death-cleanup path used by combat (so on-death effects, sacrifice checks, etc., still fire).
- **Order:** Iterate minions in (row, col) order for determinism. Tensor engine processes all minions in parallel — order doesn't matter there because no minion's burn affects another's.

Document this block verbatim in `react_stack.py` above the new hook.
</semantics_decision>

   ```python
   alive = state.minion_hp > 0  # [N, MAX_MINIONS]
   burning_mask = (state.burning_stacks > 0) & alive
   burn_damage = state.burning_stacks * BURN_DPT  # int8
   state.minion_hp = torch.where(burning_mask, state.minion_hp - burn_damage, state.minion_hp)
   state.burning_stacks = torch.clamp(state.burning_stacks - burning_mask.to(torch.int8), min=0)
   # Death cleanup: any minion whose hp dropped <= 0 from burn must be cleared
   newly_dead = burning_mask & (state.minion_hp <= 0)
   ```
