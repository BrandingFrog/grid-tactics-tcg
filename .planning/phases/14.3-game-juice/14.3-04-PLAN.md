  <action>
Replace `case 'attack':` stub with `playAttackAnimation(job, done)`.

`function playAttackAnimation(job, done)`:
1. Read `{ attackerPos, targetPos, damage, killed }` from `job.payload`.
2. Locate attacker minion DOM element. Compute pixel vector from attacker→target via `getBoundingClientRect()`. Pullback delta = `-0.3 * vector` (away from target). Strike delta = `0.7 * vector` (toward target, lands on target's edge — not center).
3. PHASE A — pullback (0-180ms): add `anim-attack-windup`, set `style.transform = translate(pullback)`. Wait 180ms.
4. PHASE B — pause (180-280ms): hold for 100ms.
5. PHASE C — strike (280-400ms): swap class to `anim-attack-strike`, set `style.transform = translate(strike)`. Wait 120ms.
6. PHASE D — impact (400ms): locate target tile DOM element, add `anim-target-hit` class. Spawn damage popup: create `<div class="damage-popup">-N</div>` (N from `job.payload.damage`), append to target tile (which must be `position: relative`; verify in CSS, add if missing). Popup auto-removes after 900ms via `setTimeout` cleanup or `animationend` listener.
7. PHASE E — return (400-700ms): clear attacker `style.transform`, swap class back to `anim-attack-windup` so the return tweens. Wait 300ms.
8. PHASE F — apply state + finish (700ms+): call `applyStateFrame(job.stateAfter)` (this will re-render and remove dead minions if `killed`). Wait one more frame, clear classes, call `done()`.

Total ≈ 800-900ms.

Update `deriveAnimationJob` to use `next.last_action` (from Task 1): when `last_action.type === 'ATTACK'`, build the attack job with `attackerPos`, `targetPos`, `damage`, `killed` directly from `last_action`. This is simpler and more reliable than diffing.
