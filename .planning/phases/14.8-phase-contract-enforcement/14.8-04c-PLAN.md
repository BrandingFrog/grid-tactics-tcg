  <how-to-verify>
**Five test scenarios — each must pass with discrete, ordered, non-racing visual beats. Test in BOTH sandbox AND a live PvP room (open two browser windows for live PvP).**

**Setup:** Server should be running locally (`python -m grid_tactics.server.app`). Open `http://localhost:5000` in browser. Use the sandbox tab for scenario reproduction; for live PvP, create a room in tab 1 and join with code in tab 2.

**Scenario 1: 3-deep react chain (acid test for spell-stage chain LIFO)**
1. In sandbox: load both decks with Acidic Rain (P1 magic), Prohibition (P2 react), Prohibition (P1 react).
2. P1 plays Acidic Rain. Observe: spell stage opens with Acidic Rain on left.
3. P2 plays Prohibition. Observe: spell stage chain extends (Acidic Rain still left, Prohibition stacks).
4. P1 plays Prohibition. Observe: chain extends again.
5. P1 passes react. Observe: chain resolves LIFO — top Prohibition resolves first, negates middle Prohibition; middle Prohibition's negate is now itself negated; Acidic Rain resolves at bottom and damages.
6. **Pass criteria:** Spell stage opens ONCE, accumulates all three cards, resolves in correct LIFO order, no jitter, no double-render, no race between chain and HP tick.

**Scenario 2: Paladin heal-on-rat-deploy (the original bug)**
1. In sandbox: P1 deploys Fallen Paladin. P2 deploys Rat.
2. End P1's turn (or use sandbox toggle to flip to P2).
3. P2 starts turn → Fallen Paladin's End-of-turn heal fires when P2 ends THEIR turn. Set up: P1 has Paladin on field, P2 has Rat on field. Use sandbox to advance turns until END-OF-TURN of P1 fires.
4. Observe sequence: rat's spell stage closes → paladin tile pulses (trigger_blip) → P1 player HP pops up `+2` → turn-2 banner appears.
5. **Pass criteria:** All FOUR beats are DISCRETE — no overlap, no skipping. The HP tick happens AFTER the spell stage closes, NOT during.

**Scenario 3: Double-paladin priority modal**
1. In sandbox: P1 has TWO Fallen Paladins on field.
2. End P1's turn. Both paladins' End-of-turn heals enqueue.
3. Observe: trigger_picker modal opens (since both belong to active player).
4. P1 picks paladin A. Observe: paladin A pulses, +2 HP pops; modal reopens.
5. P1 picks paladin B. Observe: paladin B pulses, +2 HP pops; modal closes; turn flips.
6. **Pass criteria:** Modal opens cleanly, queue pauses while open, resumes after each pick, banner fires last.

**Scenario 4: Ratchanter activated ability (conjure_rat_and_buff with pending_tutor → conjure_deploy chain)**
1. In sandbox: P1 has Ratchanter on field with mana=3.
2. P1 activates Ratchanter's ability. Observe: pending_tutor modal opens (pick a Rat from deck).
3. P1 picks a Rat. Observe: tutor modal closes, conjure_deploy modal opens (pick a tile).
4. P1 picks a tile. Observe: rat appears on tile, Ratchanter buffs (+1/+1 popup), turn continues.
5. **Pass criteria:** Two pending modals fire in sequence, each gates the queue, each releases on pick.

**Scenario 5: Multi-purpose Tree Wyrm react (opponent_ends_turn react with tutor effect)**
1. In sandbox: P1 has Tree Wyrm in hand. End P2's turn.
2. P2's End-of-turn react window opens. Observe: P1 can play Tree Wyrm as a react.
3. P1 plays Tree Wyrm. Observe: spell stage shows Tree Wyrm; react resolves; tutor modal opens for P1.
4. P1 picks tutor target. Observe: card moves to hand; spell stage closes; turn flips to P1.
5. **Pass criteria:** End-of-turn → react window → react resolution → tutor modal → tutor resolution → turn flip — all discrete, no overlap.

If ANY scenario fails, do not approve — report the failure with screenshots and the developer will fix in a follow-up to 04b. Common failure modes:
- HP tick fires DURING spell stage close (means hp_popup event isn't queueing behind react_window_closed)
- Banner fires before trigger_blip completes (means turn_flipped is shifted before trigger_blip slot completes)
- Modal opens but queue keeps draining (means slotState.pendingModalKind isn't blocking drainEventQueue)
- Spell stage chain double-opens (means react_window_opened from a chain extension is opening a new stage instead of extending)
