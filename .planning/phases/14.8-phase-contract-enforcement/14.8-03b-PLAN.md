**Modify src/grid_tactics/server/sandbox_session.py:**

`SandboxSession.apply_action(action, on_frame=None)` is currently a per-frame emit hack (commit 9c414f9). Refactor:

```python
def apply_action(self, action, on_frame=None):
    """Apply action + auto-drain.

    Returns: (final_state, events: list[EngineEvent])

    on_frame callback (legacy): if provided, called per intermediate state for back-compat.
    NEW preferred path: caller builds an EventStream, passes it via apply_action_with_events.
    """
    from grid_tactics.engine_events import EventStream
    stream = EventStream(next_seq=self._next_event_seq)
    state = self._state
    state, _ = resolve_action_capturing_events(state, action, self._library, stream)
    # Auto-PASS drain
    while not legal_actions(state, self._library):
        if state.is_game_over:
            break
        state, _ = resolve_action_capturing_events(state, pass_action(), self._library, stream)
        if on_frame is not None:
            # Legacy callback — preserve for any caller still using it; pass intermediate state
            on_frame(state)
    self._state = state
    self._next_event_seq = stream.next_seq  # M3: persist seq back to session
    return state, stream.events
```

Also add a sandbox-specific helper `apply_sandbox_edit(edit_verb, payload)` for cheat_mana / cheat_hp / set_active / undo / redo / save / load / add_card_to_zone / move_card_between_zones. Each:
- Constructs an EventStream seeded from `self._next_event_seq`
- Builds the appropriate event(s) tagged `contract_source=f"sandbox:{edit_verb}"`
- Mutates `self._state` directly (these are NOT engine actions; they bypass resolve_action and the phase-contract assertion via the sandbox: prefix shortcut from plan 01)
- Persists `self._next_event_seq = stream.next_seq` after building events
- For example: `cheat_mana` emits an EVT_MANA_CHANGE event with the new mana value.
- Returns (new_state, events).

This is what makes orchestrator decision #5 concrete: sandbox cheats produce events too, so the client uses ONE rendering pipeline.
