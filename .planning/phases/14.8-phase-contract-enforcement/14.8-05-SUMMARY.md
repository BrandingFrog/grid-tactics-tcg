# Phase 14.8 Plan 05: Strict Enforcement + Snapshot-Path Deletion + Dead-Code Cleanup Summary

**Enforcement flipped to strict in CI with soft-fail OutOfPhaseError catch; snapshot-path DOM commits deleted (engine_events is now the sole post-action wire format); playReactWindowOpened rewritten so the spell stage actually opens; last_trigger_blip + TriggerType.PASSIVE + 10 dead-code helpers DELETED (~550 lines removed across game.js + engine); all 8 CONTRACT-* requirements delivered.**

## Accomplishments

- **Strict mode live in CI** — `CONTRACT_ENFORCEMENT_MODE=strict` via conftest; production default is `shadow` (env-var override available); `off` opt-in for legacy debugging.
- **OutOfPhaseError soft-fail + M4 rollback** — `handle_submit_action` + `handle_sandbox_apply_action` both catch the error, roll state back to the pre-resolve snapshot, and emit a structured `error` socket frame with `{error_type, contract_source, phase, allowed_phases, pending_required, unknown_source, msg}`. Session never crashes.
- **Snapshot path DELETED** — post-action `state_update` / `sandbox_state` Socket.IO emits removed. `engine_events` is the sole post-action wire frame; its `final_state` field carries the authoritative snapshot for reconnect / error-recovery parity (client stashes to `window.__lastFinalState`). Initial snapshots (game_start, sandbox_create, sandbox_load, sandbox_reset, slot-load, zone edits, test-scenario load) still emit as `is_initial=True`.
- **Finding 1 fixed: spell stage actually opens now.** `playReactWindowOpened` was silently deferring to the dead snapshot-path `detectSpellCast`; rewritten to consume `slotState.lastOriginator` (stashed by preceding `playCardPlayed` / `playTriggerBlip` handlers) and call `_showSpellStage(numericId, playerIdx)` directly. Shortcut-path (no originator, zero-duration react_window_opened/closed pair) handled gracefully.
- **last_trigger_blip field DELETED** — trigger-blip flows exclusively via `EVT_TRIGGER_BLIP` on the event stream (plan 14.8-03a). Per-frame clear-at-top-of-resolve_action logic removed.
- **TriggerType.PASSIVE DELETED** — value 5 BURNED (not reused). CardLoader raises at load time on any card JSON that still carries `"trigger": "passive"`. Regression guard test added.
- **3 engine dead-code helpers deleted** — `_fire_passive_effects`, `_enqueue_dead_minions_and_cleanup_zones`, `_drain_pending_death_queue`. All were no-ops since Phase 14.7-03 / 14.7-05b.
- **7 client dead-code helpers deleted** — `derivePlayerHpDeltaAnims`, `detectReactWindowClose`, `detectSpellStageClose`, `detectSpellCast`, `deriveCardFlyJobs`, `_deriveFlyForPlayer` (+ its `_multisetDelta` helper), `deriveDrawJobs`, `deriveAnimationJob`. ~550 lines net deletion in game.js.
- **pending_death_queue field DELETED** — defensive no-op since 14.7-05b; last users (the two engine helpers) deleted above.
- **CONTRACT-01 through CONTRACT-08 all marked [x]** in REQUIREMENTS.md.
- **Net code reduction: +278 insertions / −785 deletions = −507 lines.**

## Decisions Made

- **Snapshot path handling (plan called for DELETE, how to do it):** Server-side DELETED for post-action; client-side handlers KEPT alive for reconnect safety. `window.__lastFinalState` still stashes on every engine_events frame. This mirrors the spirit of the plan while keeping the reconnect safety net functional.
- **Zone-edit / undo / redo / reset emit posture:** Treat as `is_initial=True` (wholesale state replacement, not animated) so the client snaps to the new state without an events-driven replay.
- **TriggerType.PASSIVE deletion strategy:** Burn value 5, don't reuse — preserves tensor-engine integer encoding stability across the migration (append-only invariant honored even across deletions).
- **Finding 1 fix side (client vs server):** Client-side (`slotState.lastOriginator` stash) — simpler, no engine payload changes, respects plan 03a event shape boundaries.
- **`OutOfPhaseError` catch posture:** Soft-fail (orchestrator decision #2): rollback + structured error emit + session continues. Never crash the session.
