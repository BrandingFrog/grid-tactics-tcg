## Decisions Made

All key decisions captured in frontmatter `key-decisions`. Core dispositions:

1. **Per-call EventStream pattern.** Identical for live PvP (handle_submit_action) and sandbox (SandboxSession.apply_action / apply_sandbox_edit). Each call seeds `EventStream(next_seq=session.next_event_seq)`, threads it through every nested resolve_action, persists `stream.next_seq` back. Plan 04b's lastSeenSeq dedup depends on this monotone counter.

2. **ONE EventStream per submit_action / apply_action call** — even when the chain composes multiple resolve_action calls (auto-advance loop, drained PASSes). Single seq-ordered stream covers the whole user-visible chain.

3. **9c414f9 per-frame emit hack decommissioned.** Sandbox auto-drain produces ONE engine_events socket frame per apply_action call regardless of how many auto-PASSes drain. Pacing moves from socket-frame cadence to event animation_duration_ms (plan 04a/b client consumption). The transient-signal regressions that the hack worked around are handled by dedicated event types from plan 03a.

4. **Sandbox edit verbs flow through apply_sandbox_edit** with `contract_source='sandbox:<verb>'`. Bypasses phase-contract assertions (sandbox: prefix shortcut from plan 01) but produces events for the client's uniform rendering pipeline. Orchestrator decision #5 made concrete.

5. **filter_engine_events_for_viewer uses explicit None checks, NOT `or`-chains** for owner-key resolution. P1 = 0 is falsy in Python — `or` would treat P1 ownership as "no owner set" and apply conservative redaction even to the picker themselves. Documented as a footgun comment.

6. **god_mode bypass** in filter_engine_events_for_viewer returns input unchanged (no copy). Used by sandbox (`_emit_sandbox_state` always passes god_mode=True) and spectator-god (`_fanout_state_to_spectators` reads `slot.god_mode`).

7. **Dual-emit pattern for back-compat.** Both `state_update`/`sandbox_state` (legacy snapshot) AND `engine_events` (new event stream) fire from the same handler. Plan 14.8-05 drops the legacy paths once all clients have switched. Old clients (pre-04a) ignore the new frame and continue rendering from the snapshot — zero breakage.

8. **Save/load handlers NOT refactored** to apply_sandbox_edit. Saves are side-effect-only (write a file, don't change state). Loads use `load_dict` which already resets the seq counter so the next event stream is fresh from the client's POV. The 16-handler count from the plan is a soft target — 14 verbs are covered by apply_sandbox_edit, 2 are intentional pass-throughs.
