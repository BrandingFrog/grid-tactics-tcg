# Phase 14.5 Plan 04: Uniform Card Renderer Summary

**One-liner:** Collapsed `renderHandCard` and `renderDeckBuilderCard` into a single shared `renderCardFrame(c, opts)` HTML builder so hand, deck builder, tooltip preview, and future pile modals render from one source of truth.

## What shipped

### New API: `renderCardFrame(c, opts)`

Single full-size YGO-style card HTML builder. Returns a string.

```
opts = {
  context:         'deck-builder' | 'hand' | 'tooltip' | 'pile',
  count:           number,  // deck-builder badge; -1 = prohibited, undefined = no badge
  handIndex:       number,  // stamps data-hand-idx for click delegation
  numericId:       number,  // stamps data-numeric-id
  dim:             bool,    // adds .card-dimmed (hand can't-afford/not-my-turn)
  showReactDeploy: bool,    // deck-builder shows '▶ Deploy' react hint; hand suppresses
}
```

### Wrappers

- `renderDeckBuilderCard(numericId, count)` → 7-line wrapper, `showReactDeploy: true`.
- `renderHandCard(numericId, handIndex, currentMana, isMyTurn)` → 12-line wrapper,
  computes `dim = !(canAfford && isMyTurn)`, `showReactDeploy: false`.

### Tooltip preview path

Already routed through `renderDeckBuilderCard(numericId, -1)` at
`showCardTooltip()` (deck-builder-tooltip) and `showGameTooltip()`
(in-game hover). Both now automatically share the same renderer —
no callsite changes needed.

## Context class strategy (zero visual drift)

Hand cards are stamped with BOTH `.card-frame-full` (base layout: 192px width,
flex column, 58% art height) AND `.card-frame-hand` (context hook). This means:

- All existing hand state selectors keep working unchanged:
  `.card-frame-hand.card-playable`, `.card-frame-hand.card-selected-hand`,
  `.card-frame-hand.card-react-playable`, and the mobile width override
  `@media (max-width:600px) { .card-frame-hand { width: 132px } }`.
- The dead `.card-frame-hand { width:192px; display:flex; flex-direction:column }`
  and `.card-art-hand { height:58% }` base rules were deleted — they were
  verbatim duplicates of `.card-frame-full` / `.card-art-full` and re-declaring
  them risked silent drift.
- `.card-art-hand` CSS class is still stamped on hand art by `renderCardFrame`
  as a no-op hook for potential future hand-specific art tweaks.

## Decisions Made

- **Single HTML builder + thin wrappers** instead of branching inside a single
  `renderHandCard`/`renderDeckBuilderCard`. Wrappers keep callsites stable and
  document per-context intent (showReactDeploy, dim derivation).
- **Hand keeps `.card-frame-hand` class** — deleting it would have required
  rewriting 4+ state selectors, the playable-glow CSS, selection state, react
  glow, and mobile sizing. Keeping it as a pure state-modifier hook was lower
  risk with identical visual outcome.
- **`showReactDeploy` flag** — the original `renderHandCard` suppressed the
  "▶ Deploy" hint line that `renderDeckBuilderCard` showed for multi-purpose
  react cards (e.g. Dark Sentinel). Preserving that asymmetry via an explicit
  opt-in flag rather than silently flipping behavior.
- **Task 2 visual smoke test deferred to post-deploy Playwright E2E** — same
  posture as 14.1-04 / 14.2-04 / 14.3-01 / 14.3-04 / 14.3-07 / 14.4-05.
  Structural code review confirms: data attributes preserved, context classes
  preserved, dim logic preserved, tooltip path preserved (was already calling
  `renderDeckBuilderCard`), mobile media query still valid.
