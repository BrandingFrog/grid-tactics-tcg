```python
def fire_start_of_turn_triggers(state: GameState, library: CardLibrary) -> GameState:
    """Fire all ON_START_OF_TURN triggered effects for the current active player's minions.

    Ordering: (row, col) for determinism (14.7-05 replaces with priority queue + modal).
    Fires AFTER tick_status_effects (burn ticks) — burn is a status tick, not a trigger.
    Fizzle (14.7-06) not yet implemented — effects resolve blindly against their target.

    Only fires effects OWNED by the active player (card's owner). An enemy minion's
    Start: trigger would not fire at your turn start — it fires at ITS owner's turn start.
    """
    from grid_tactics.effect_resolver import resolve_effect
    from grid_tactics.action_resolver import _check_game_over, _cleanup_dead_minions

    active_side = state.players[state.active_player_idx].side

    ordered_ids = [
        m.instance_id
        for m in sorted(state.minions, key=lambda m: (m.position[0], m.position[1]))
    ]
    for inst_id in ordered_ids:
        m = state.get_minion(inst_id)
        if m is None:
            continue
        if m.owner != active_side:
            continue
        card_def = library.get_by_id(m.card_numeric_id)
        for effect in card_def.effects:
            if effect.trigger != TriggerType.ON_START_OF_TURN:
                continue
            state = resolve_effect(
                state, effect, m.position, m.owner, library, target_pos=None,
            )

    state = _cleanup_dead_minions(state, library)
    state = _check_game_over(state)
    return state


def fire_end_of_turn_triggers(state: GameState, library: CardLibrary) -> GameState:

    Same ordering / ownership / fizzle caveats as fire_start_of_turn_triggers.
```

```python
def enter_end_of_turn(state: GameState, library: CardLibrary) -> GameState:
    """Enter END_OF_TURN phase: fire End: triggers → open react window.

    When the end-of-turn react window closes, close_end_react_and_advance_turn
    runs _close_end_of_turn_and_flip + enter_start_of_turn for the next player.
    """
    state = replace(state, phase=TurnPhase.END_OF_TURN)

    # Fire ON_END_OF_TURN triggers
    state = fire_end_of_turn_triggers(state, library)
    if state.is_game_over:
        return state

    # Open end-of-turn react window
    state = replace(
        state,
        phase=TurnPhase.REACT,
        react_player_idx=1 - state.active_player_idx,  # opponent reacts
        react_context=ReactContext.BEFORE_END_OF_TURN,
        react_return_phase=TurnPhase.END_OF_TURN,
    )
    return state
```

Step 4 — Where is enter_end_of_turn called FROM? In 14.7-02 it's called at the end of the turn, but that wiring isn't complete yet. Today when a player finishes their ACTION phase (resolved all post-action react windows), the engine advances the turn directly via _close_end_of_turn_and_flip + enter_start_of_turn for the new active player.

After this plan: when a turn's ACTION phase ends (player has no more legal actions and fatigue-bleed auto-passes, OR they explicitly PASS), the state should transition to END_OF_TURN BEFORE the active-player flip. Find the call site where today's code transitions from REACT → ACTION after an action's react window closes. That's resolve_react_stack's `return_phase == TurnPhase.ACTION` branch (14.7-02 Task 2 Step 3).

Update that branch: the CURRENT behavior is `return close_end_react_and_advance_turn(state, library)`. We need to understand what "after-action react window closed" means:
- If the player still has more actions on their turn → they should go back to ACTION (not advance turn).
- If the player has NO MORE ACTIONS (fatigue, or they PASSED) → advance turn.

Today's engine uses PASS as the turn-ending action (D-16). Fatigue bleed auto-passes when no legal actions exist. The distinction between "stay in ACTION for more turns" and "advance turn" today is: PASS's react window resolves → advance turn. Any OTHER action's react window resolves → advance turn too (because our game is one-action-per-turn).

So the current model: every action, regardless of type, advances the turn after its react window closes. We'll preserve that.

**Redistribute the turn-advance logic:**
- `resolve_react_stack` with `return_phase=TurnPhase.ACTION` should now invoke `enter_end_of_turn(state, library)` INSTEAD OF `close_end_react_and_advance_turn`. Rationale: the after-action react window ends → the active player's turn is over → enter END_OF_TURN (fires End: triggers + opens end-of-turn react window).
- `resolve_react_stack` with `return_phase=TurnPhase.START_OF_TURN` → transitions to ACTION (14.7-02 behavior preserved).
- `resolve_react_stack` with `return_phase=TurnPhase.END_OF_TURN` → calls `close_end_react_and_advance_turn` (which in turn calls _close_end_of_turn_and_flip + enter_start_of_turn for new active player).

Update the resolve_react_stack dispatch table accordingly:

```python
if return_phase == TurnPhase.ACTION:

elif return_phase == TurnPhase.START_OF_TURN:
```

Step 5 — Special case: pre-14.7 tests that expect turn-advance after a single action without any end-of-turn detour. Those tests will break because there's now an End-of-Turn react window between the action's react close and the turn advance. If the end-of-turn phase has NO triggers to fire AND no minions with End: effects, the end-of-turn react window opens with an empty react_stack, and with no legal react cards in hand it will close immediately via auto-PASS × 2 in the events.py loop.

But for PURE UNIT TESTS that call resolve_action then inspect state directly (not going through events.py auto-pass loop), the state will be stuck in REACT with react_context=BEFORE_END_OF_TURN.

Decision: unit tests that need a full turn cycle should call a new helper `advance_to_next_turn(state, library)` that drives PASS-PASS through any open react window until phase=ACTION with new active player. Add this helper in react_stack.py:

```python
def advance_to_next_turn(state: GameState, library: CardLibrary) -> GameState:
    """Test helper: drive PASS-PASS through any open react/phase until ACTION for next player."""
    from grid_tactics.actions import Action
    from grid_tactics.enums import ActionType
    start_turn = state.turn_number
    safety = 0
    while state.turn_number == start_turn and not state.is_game_over:
        safety += 1
        if safety > 20:
            raise RuntimeError("advance_to_next_turn safety exceeded")
        if state.phase == TurnPhase.REACT:
            state = handle_react_action(state, Action(action_type=ActionType.PASS), library)
        elif state.phase == TurnPhase.START_OF_TURN:
            state = enter_start_of_turn(state, library)
        elif state.phase == TurnPhase.END_OF_TURN:
            state = enter_end_of_turn(state, library)
        else:
            return state  # in ACTION, nothing more to drive
```
