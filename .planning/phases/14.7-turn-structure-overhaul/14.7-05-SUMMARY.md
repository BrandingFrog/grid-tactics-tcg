# Phase 14.7 Plan 05: Simultaneous-Trigger Priority Queue + Modal Picker Summary

**Simultaneous START_OF_TURN / END_OF_TURN triggers now enqueue into turn/other priority queues instead of resolving inline. Turn player's queue drains first. 2+ simultaneous triggers in a queue open a modal card-picker reusing the existing tutor modal (renderDeckBuilderCard). Each picked/auto-resolved trigger opens its own REACT window. Drain-recheck hook in resolve_react_stack ensures cascading triggers fully drain before the phase dispatch advances the turn.**

## Accomplishments

### Task 1: Data model + serializer

- **ActionType** extended with `TRIGGER_PICK = 17` and `DECLINE_TRIGGER = 18` (append-only so existing slots 0..16 stay stable for the tensor action encoder).
- **PendingTrigger** frozen dataclass in `game_state.py` captures everything needed to resolve a queued trigger after its source minion may have moved / died: `trigger_kind`, `source_minion_id`, `source_card_numeric_id`, `effect_idx`, `owner_idx`, `captured_position`, `target_pos`.
- **GameState** gains 3 fields: `pending_trigger_queue_turn`, `pending_trigger_queue_other`, `pending_trigger_picker_idx`. Fully serialized by `to_dict` / `from_dict` with backward-compat fallback (old dicts missing the keys reconstruct with empty defaults).
- **Sandbox save/load** round-trips all three fields end-to-end (file-based slot persistence).

### Task 2: Engine rewiring + drain + handlers + drain-recheck

- **`_enqueue_turn_phase_triggers`** (new helper in `react_stack.py`): collects simultaneous ON_START_OF_TURN / ON_END_OF_TURN effects for the active player's minions in (row, col) order, splits them into turn-queue / other-queue by owner, and pushes them onto the existing queues (appended, not replaced, so cascading phases don't clobber in-flight entries).
- **`fire_start_of_turn_triggers` / `fire_end_of_turn_triggers`** rewritten: they now enqueue instead of resolve inline, then call `drain_pending_trigger_queue`.
- **`drain_pending_trigger_queue`** implements spec §7.2 priority:
  - Turn queue drains first (priority).
  - `>=2` entries in current queue → set `pending_trigger_picker_idx` (opens modal).
  - Exactly 1 entry → auto-resolve via `_resolve_trigger_and_open_react_window`.
  - 0 entries → fall through to the other queue; both empty exits.
- **`_resolve_trigger_and_open_react_window`** is the single pop-and-open helper used by both the auto-resolve path and the `TRIGGER_PICK` handler. It resolves the effect using `captured_position` (SELF_OWNER targeting still works even if the source minion has moved), pops the entry from the correct queue, and opens a REACT window tagged with `AFTER_START_TRIGGER` / `BEFORE_END_OF_TURN` depending on `trigger_kind`. Preserves existing `react_return_phase` if already set so the final phase-dispatch returns to the correct phase.
- **`_apply_trigger_pick`** moves the picked entry to the front of the queue before calling `_resolve_trigger_and_open_react_window`. This preserves single-source-of-truth pop semantics (the helper always pops index 0) and prevents a double-pop bug that I caught on first test run.
- **`_apply_decline_trigger`** clears the picker's entire remaining queue + closes the modal + re-enters drain for the other queue.
- **Drain-recheck hook in `resolve_react_stack`** — the critical ordering constraint:
  ```
  LIFO loop → _cleanup_dead_minions → is_game_over → DRAIN-RECHECK → return_phase dispatch
  ```
  When a trigger resolution leaves entries in either queue, re-enter drain (opening the next window / modal) instead of advancing the turn. The `test_two_triggers_second_fires_after_first` regression test specifically validates this ordering.
- **Legal actions** gate: `_pending_trigger_picker_actions` enumerates one `TRIGGER_PICK(queue_idx=i)` per queue entry plus one `DECLINE_TRIGGER`. Non-picker gets empty tuple (opponent waits). Gate placed BEFORE the REACT-phase enumeration so a closing trigger window doesn't surface legal react cards.
- **RL action space** (`rl/action_space.py`) extended to encode/decode `TRIGGER_PICK` (reuses `PLAY_CARD` slot space [0:250] with `stride=GRID_SIZE`, same as `TUTOR_SELECT`) and `DECLINE_TRIGGER` (reuses `PASS` slot 1001). Decode priority: trigger_picker > conjure > tutor > post_move (phase-agnostic gate checked first).
- **`_cleanup_dead_minions` UNCHANGED** per scope fence. Death-trigger migration lives in 14.7-05b.

## Drain-Recheck Ordering Constraint (Detail)

The drain-recheck hook in `resolve_react_stack` is the load-bearing correctness guarantee for multi-trigger sequences. Its placement is specifically:

```
1. LIFO loop resolves react_stack entries (reactions, negates, originators)
2. _cleanup_dead_minions runs (may enter pending_death_target)
3. is_game_over check — early return if over (skip drain)
4. ** DRAIN-RECHECK (14.7-05) ** — re-enter drain if queues non-empty
5. return_phase dispatch (advance to ACTION / END_OF_TURN / flip turn / etc.)
```

If the drain-recheck runs AFTER the dispatch, the dispatch would:
- Close the trigger #1 react window.
- Advance to `enter_end_of_turn` (because `react_return_phase == ACTION`).
- Leave trigger #2 stranded in `pending_trigger_queue_turn` until the NEXT time the queue is touched (possibly never, for a short game).

If the drain-recheck runs BEFORE `_cleanup_dead_minions`, a trigger that killed its source could fizzle before cleanup runs (and we'd miss on-death effects).

The comment block in `resolve_react_stack` states this ordering explicitly. The regression test `test_two_triggers_second_fires_after_first` drives two simultaneous Fallen Paladin heals through the queue: pick one, PASS-PASS the window, and assert both paladins healed. Without the drain-recheck at the correct position, only one heals and the test fails loudly.

## Decisions Made

- **Scope-fenced to START_OF_TURN + END_OF_TURN only** — Death: migration (_cleanup_dead_minions) is separate plan 14.7-05b. The user explicitly chose option (a) full migration for Death:, but the blast radius warranted splitting.
- **Singleton auto-resolves** without modal — spec §7.2 only mandates the modal for 2+. Saves a click in the common case.
- **DECLINE_TRIGGER clears entire remaining queue** (fizzle-all), not skip-one.
- **`captured_position` field on PendingTrigger** — survives minion movement/death. True fizzle re-validation lands in 14.7-06.
- **Picker gate BEFORE REACT-phase delegation** — modal pre-empts react-phase enumeration. Only picker owner can act.
- **Picker modal reuses renderDeckBuilderCard** — user directive. No new modal primitives.
- **Turn queue drains before other queue** — spec §7.2 turn-player priority.
