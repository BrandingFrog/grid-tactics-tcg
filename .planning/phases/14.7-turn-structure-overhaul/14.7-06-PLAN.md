<objective>
Implement the fizzle rule from spec §7.3: an effect whose target is no longer valid at resolution time fizzles silently (no error, no partial resolution). This enables the strategic pre-empt described in spec §7.4 (RGB Laser Cannon vs Giant Rat example).

Purpose: Today effect resolution blindly operates on captured (target_pos, instance_id) without re-validating at resolve time. That's fine for most cases but BREAKS the simultaneous-deaths worked example. Per spec §7.4, the turn player MUST be able to destroy an enemy minion with their own death effect BEFORE that enemy minion's death effect resolves, causing the enemy's target to fizzle.

This is a minimal plan: one helper + gated check in resolve_effect + modified _resolve_trigger_and_open_react_window + tests.

Output:
- _validate_target_at_resolve_time(state, effect, source_minion_id, target_pos) helper in effect_resolver.py
- resolve_effect early-returns (returns state unchanged) when validation fails for SINGLE_TARGET / ADJACENT / SELF_OWNER
- _resolve_trigger_and_open_react_window calls validation before effect resolution; if fizzle, pop trigger without opening react window (the react window is meaningless for a no-op)
- Tests for each fizzle case: missing target, wrong-faction target, dead source for ADJACENT/SELF_OWNER
- Worked example §7.4 passes as integration test
</objective>

```python
def _validate_target_at_resolve_time(
    state: GameState,
    effect,  # EffectDefinition
    source_pos: tuple[int, int] | None,
    source_minion_id: Optional[int],
    target_pos: Optional[tuple[int, int]],
) -> bool:
    """Return True if the effect can still resolve given current state.

    Per spec §7.3, an effect fizzles when:
      - SINGLE_TARGET: no minion exists at target_pos
      - ADJACENT: source minion is no longer alive (can't compute adjacency)
      - SELF_OWNER when source is a minion (not a magic/player): source minion no longer alive

    OPPONENT_PLAYER never fizzles.
    Area effects (ALL_ENEMIES, ALL_ALLIES, ALL_MINIONS) naturally no-op on empty
    target lists — don't need this check (handled in their respective handlers).
    """
    from grid_tactics.enums import TargetType

    tt = effect.target_type

    if tt == TargetType.SINGLE_TARGET:
        if target_pos is None:
            return False
        minion_id = state.board.get(target_pos[0], target_pos[1])
        if minion_id is None:
            return False
        m = state.get_minion(minion_id)
        if m is None or m.current_health <= 0:
            return False
        return True

    if tt == TargetType.ADJACENT:
        # ADJACENT effects need a live source minion (adjacency computed from it)
        if source_minion_id is not None:
            src = state.get_minion(source_minion_id)
            if src is None or src.current_health <= 0:
                return False
        elif source_pos is None:
            return False
        return True

    if tt == TargetType.SELF_OWNER:
        # If source is a MINION (identified via source_minion_id), that minion
        # must still be alive. For magic-card casts, source_minion_id is None
        # and SELF_OWNER refers to the casting player — always valid.
        if source_minion_id is not None:
            src = state.get_minion(source_minion_id)
            if src is None or src.current_health <= 0:
                return False
        return True

    # OPPONENT_PLAYER / ALL_ENEMIES / ALL_ALLIES / ALL_MINIONS — never fizzle via this helper
    return True
```
