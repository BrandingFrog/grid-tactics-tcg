## Deviations from Plan

### Plan referenced files that didn't exist

1. **tests/test_room_manager.py** — the plan treats this as an existing file ("Use existing test fixtures / helpers in the file for CardLibrary loading"). It does not exist. Created fresh. Followed `test_pvp_server.py`'s `CardLibrary.from_directory(Path("data/cards"))` pattern for library loading. No shared fixture was necessary — tests are small and instantiate RoomManager directly.
2. **tests/test_events.py** — the plan treats this as an existing file ("Use the existing flask-socketio test client pattern in test_events.py"). It does not exist. Created fresh and mirrored the exact socketio.test_client + register_events + get_received pattern from `test_pvp_server.py`. Added to conftest collect_ignore_glob so dev environments without flask_socketio stay green.

Neither file creation triggered a Rule 1-4 deviation — they are the direct deliverables of the plan, just without the "existing file" assumption.

### Intentional scope choices

1. **Task 2 state_update test** issues a PASS from whichever of Alice/Bob is the active player by trying both in sequence. This sidesteps needing to parse `your_player_idx` from game_start, which in the test harness depends on a coin flip in `start_game`. Same effect, simpler test.
2. **Task 2 non-god filter assertion** uses tolerant shape-matching: `p1.get("hand") in (None, [], ()) or "hand_count" in p1`. `filter_state_for_player` strips the opponent hand to a count; accepting both "hand missing" and "hand_count present" is robust against small schema drift in the filter.
3. **Task 4 deferral** documented in-summary and in STATE.md Decisions, matching the precedent set by every prior 14.x closeout.
