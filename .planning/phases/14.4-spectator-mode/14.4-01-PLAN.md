  <action>
  Modify room_manager.py:

  1. Add `role` concept. Simplest: store spectators as a dict `spectators: dict[str, SpectatorSlot]` on WaitingRoom keyed by session_token, where SpectatorSlot is a small dataclass with `sid: str`, `display_name: str`, `god_mode: bool`. Also add `spectators: dict = field(default_factory=dict)` to WaitingRoom.

  2. Add a parallel internal index `self._token_role: dict[str, str]` on RoomManager mapping session_token -> 'player' | 'spectator'. Populate it everywhere a token is currently created (create_room, join_room) with 'player'. Add a helper `get_role(token) -> str | None`.

  3. Add `self._token_to_room: dict[str, str]` if not already present (check existing impl — there's already a token->room lookup for players; spectators need to participate in it). Update spectator join to register tokens in the same room-code lookup so get_room_code_by_token works for spectators too.

  4. Add method `join_as_spectator(self, room_code: str, display_name: str, sid: str, god_mode: bool) -> tuple[str, WaitingRoom]`:
     - Look up room (via waiting rooms OR active games — spectators can join mid-game; if no waiting room, find via game session's room_code). Raise ValueError("Room not found") if neither exists.
     - Generate a fresh session_token (reuse the same uuid scheme used by join_room).
     - Construct SpectatorSlot and add to room.spectators[token].
     - Register token in _token_role as 'spectator', token->room lookup, sid->token lookup.
     - Return (token, room).

  5. Add `get_spectator_tokens(self, room_code: str) -> list[str]`: returns list of spectator tokens for the room (works for both waiting and in-game rooms).

  6. Add `remove_spectator(self, token: str) -> str | None`: removes spectator from room and indexes. Returns room_code or None. Wire this into the existing disconnect path (find where sid disconnect cleans up players and add a branch for spectators using _token_role).

  7. IMPORTANT: When a WaitingRoom transitions into a GameSession via start_game, the spectators dict must survive. If WaitingRoom is replaced by GameSession, store spectators on GameSession too (add `spectators: dict = field(default_factory=dict)` to GameSession in game_session.py only if needed — otherwise keep spectators keyed by room_code in RoomManager itself: `self._room_spectators: dict[str, dict[str, SpectatorSlot]]`). Pick whichever is least invasive; document the choice in a top-of-file comment.

  Do NOT touch events.py, view_filter.py, or frontend in this plan.
