## Accomplishments

- **Toolbar markup:** four-row control surface inside #sandbox-toolbar covering search/zone/player selection, control toggle + 9 action buttons + history pill, 6 cheat inputs (HP / current_mana / max_mana x P1/P2), and a Server Saves panel with slot name input + Save + Refresh + dynamic slot list.
- **Emit-gate reuse:** `submitAction` at game.js:3328 gained a single `if (sandboxMode) socket.emit('sandbox_apply_action', actionData); else socket.emit('submit_action', actionData);` conditional between SANDBOX-EMIT-GATE-START (line 3331) and SANDBOX-EMIT-GATE-END (line 3337). Every onHandCardClick / onBoardCellClick / onMinionClick code path now routes sandbox clicks through sandbox_apply_action without any other changes. No accessor shims were added.
- **Move-to popover:** `makeSandboxMoveButton` + `openSandboxMovePopover` helpers inside the SANDBOX section emit sandbox_move_card with `{player_idx, card_numeric_id, src_zone, dst_zone}`. The popover auto-closes on outside click.
- **Pile modal additive injection:** `showPileModal(title, cardNumericIds, sandboxCtx?)` at game.js:1884 gained an optional third arg. When `sandboxMode && sandboxCtx` is true, the per-cell loop (game.js:1908) appends `makeSandboxMoveButton(sandboxCtx.playerIdx, nid, srcZone)` where srcZone is derived from sandboxCtx.pileType ('graveyard'|'exhaust'|'deck_top'). Live-game pile modal calls (which pass only 2 args) are byte-unchanged.
- **Hand row additive injection:** `renderHand` at game.js:4850 gained an `if (sandboxMode && typeof makeSandboxMoveButton === 'function')` block at game.js:4894 inside the existing per-card append loop. It appends a Move-to button as a sibling of the card element. The play-from-hand click handler is unchanged.
- **Sandbox pile access:** `renderSandboxStats` now emits `sandbox-pile-btn` buttons per player (grave / exhaust / deck) that open the existing pile modal with sandboxCtx. No #screen-game pile button duplication. Buttons are wired via post-innerHTML querySelectorAll.
- **Cheat inputs (DEV-06):** blur + Enter only. `addEventListener('blur', commit)` + `addEventListener('keydown', ...)` with `input.blur()` on Enter. Never `addEventListener('input', ...)`. Values emit sandbox_set_player_field.
- **Deck import (DEV-03):** reuses `loadDeckSlots()` at game.js:780 and `getDeckTotal()` at game.js:1035 (existing helpers). Prompt-based slot picker -> target player prompt -> expands `{numericId: count}` map into flat list -> emits sandbox_import_deck.
- **Share code (DEV-07):** sandboxEncodeShareCode uses `new TextEncoder().encode(JSON.stringify(stateDict))` -> String.fromCharCode -> btoa. sandboxDecodeShareCode is the reverse via `Uint8Array.from(binary, c => c.charCodeAt(0))` -> TextDecoder.decode -> JSON.parse. **NEVER** escape / unescape.
- **localStorage autosave (DEV-07):** every sandbox_state frame writes `{state, active_view_idx}` to SANDBOX_AUTOSAVE_KEY = 'gt_sandbox_autosave_v1'. initSandboxScreen restores via sandbox_load on screen activation.
- **Save file download:** sandbox_save_blob listener creates a Blob, URL.createObjectURL, clicks a synthetic anchor, and revokes the URL. Filename is `sandbox-<ISO-timestamp>.json`.
- **Server saves panel (DEV-08):** renderSandboxSlotList renders one row per slot with Load / Delete buttons. Delete button confirms via native confirm(). sandbox_slot_list listener refreshes the panel. sandbox_slot_saved clears the slot name input. sandbox_slot_deleted is a no-op because sandbox_slot_list refresh handles the UI.
- **Undo/Redo (DEV-09):** Undo / Redo buttons emit sandbox_undo / sandbox_redo. renderSandboxToolbarState disables them when sandboxUndoDepth / sandboxRedoDepth is 0.
- **Control toggle (DEV-05):** Controlling: P1 / P2 buttons emit sandbox_set_active_player. The server mutates state.active_player_idx; the LAYOUT does NOT change (P1 stays top, P2 stays bottom). No flip / view-toggle / perspective-swap UI was added -- verified by `awk '/SANDBOX-SECTION-START/,/SANDBOX-SECTION-END/' game.js | grep -iE 'flip|view-toggle|perspective.swap'` -> empty.
- **Toolbar state sync:** renderSandboxToolbarState is called from inside the sandbox_state handler after gameState/myPlayerIdx/legalActions are assigned. It updates history pill text, toggles control-button active class, disables undo/redo when depths are 0, and syncs cheat input values via `.value = String(player[field])` -- skipping any input that is `document.activeElement` to avoid clobbering user typing (which would also immediately re-fire blur in a feedback loop).

## Decisions Made

- **Pile access via sandbox-stats buttons:** The plan's verify steps reference "open the graveyard pile modal", but the existing pile buttons live on #screen-game only. Rather than adding a parallel pile button bar, I embedded per-player pile-open buttons directly into renderSandboxStats. This reuses the entire Phase 14.5 pile modal (showPileModal + renderCardFrame) and keeps the sandbox UI self-contained.
- **showPileModal extension over fork:** The modal was updated to accept an optional `sandboxCtx` third arg. All existing live-game callers (4 in setupPileHandlers) continue to call it with 2 args. Zero regression risk.
- **renderSandboxStats pile-button wiring via querySelectorAll:** After the innerHTML rewrite, a single pass binds click handlers. This avoids inline `onclick=` attrs (which would require escaping) and survives every state frame cleanly.
- **initSandboxScreen skips sandbox_create when restoring:** The handle_sandbox_load handler auto-creates the session if missing AND emits sandbox_card_defs. So the init flow simplifies to "try sandbox_load, else sandbox_create", not "create then load".
- **escapeHtml defined inside SANDBOX section:** No existing escapeHtml helper in game.js. Rather than introduce a module-level utility, the helper is scoped to the sandbox section where search-result innerHTML is the only consumer.
- **Module-level SANDBOX_ALL_ZONES constant:** The list of ALL_ZONES used by the Move-to popover is a top-level `var` so both the popover opener and future sandbox extensions can reuse it. Prefix-scoped to SANDBOX_ALL_ZONES to avoid accidental collision.
