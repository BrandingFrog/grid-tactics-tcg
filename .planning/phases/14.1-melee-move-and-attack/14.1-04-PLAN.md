```css
}
```

  <files>src/grid_tactics/server/static/game.js</files>
  <action>
1. In `src/grid_tactics/server/static/game.js` (around lines 2061-2135 for interaction state, ~2265-2310 for highlightBoard):

2. Add a new interaction mode value: `interactionMode = 'post_move_attack_pick'`. Enter it automatically when the server state arrives with `pending_post_move_attacker_id != null`. Exit it when the field becomes null (after attack or decline).

3. In `highlightBoard()`:
   - When in `post_move_attack_pick` mode: read `state.pending_attack_range_tiles` and add class `attack-range-footprint` to every matching `.tile` element.    - Clear both classes when leaving the mode.

4. Click handler: when in `post_move_attack_pick` mode and the player clicks a tile in `pending_attack_valid_targets`, send an ATTACK action via the existing WS sender with `attacker_id = pending_post_move_attacker_id` and `target` = the clicked enemy's minion id. Clicks on other tiles are ignored (do not deselect or exit mode — only attack or decline can exit).

5. Decline button: add a floating action button (or HUD element) that ONLY appears when `interactionMode === 'post_move_attack_pick'`. Label: "Decline Attack" (or "Skip Attack"). On click, send `{ action_type: 'DECLINE_POST_MOVE_ATTACK' }` (numeric value 8) via the WS action sender. Style it so it's obvious — top-right of the board area, red border, not hidden in a menu.

6. Make sure the existing "click minion → highlight forward move tile → click to move" flow remains unchanged for ALL minions. The only change: after the move action's response arrives with `pending_post_move_attacker_id` set, the client automatically enters attack-pick mode without an extra click.

7. Ranged minions: their move flow is unchanged. Server will not set pending for them, so the client never enters attack-pick mode. Verify this by reading `pending_post_move_attacker_id == null` in their post-move state.
