**Net result:** The auto-attack-after-move behavior is gone from every layer (Python engine, tensor engine, legal-action mask, server payload, browser UI). Melee minions now move first, then receive a single pending question — pick a valid in-range enemy or decline — and the react window fires once after the pending state clears. Ranged minions retain their existing single-action attack semantics. The action-space layout `[0:1262]` is unchanged; slot 1001 is reinterpreted as `DECLINE_POST_MOVE_ATTACK` only when a pending attacker is set.

**Known follow-up (non-blocking):** RL checkpoints trained pre-14.1 learned the old auto-attack behavior and are now suboptimal. They still load and play; a retrain pass is queued for a future phase.
