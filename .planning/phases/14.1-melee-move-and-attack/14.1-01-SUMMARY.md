# Phase 14.1 Plan 01: Python Engine Pending Post-Move Attack Summary

**One-liner:** Replaced auto-attack-after-move in the Python engine with a player-driven pending-post-move-attack state for melee minions, gated through `resolve_action` so move+attack/decline counts as one logical action with a single react window.

## Encoding Decision (locked in)

Action-space layout `[0:1262]` is **PRESERVED** — no new slots added.

- New `ActionType.DECLINE_POST_MOVE_ATTACK = 8` exists in Python only.
- It is encoded by **reusing slot 1001 (PASS)** when `state.pending_post_move_attacker_id is not None`. In pending state, slot 1001 means "decline post-move attack" instead of "pass turn"; PASS-as-end-turn is illegal in pending state.
- Post-move ATTACK reuses existing ATTACK[350:975] slots; legal_actions (Wave 3) will mask to in-range targets from the pending attacker only.
- Tensor parity (Wave 2): `pending_post_move_attacker_id` becomes `[N]` int tensor with sentinel `-1`.

**Impact on RL checkpoints:** Existing checkpoints become stale (they learned the old auto-attack model) but remain loadable. Documented as known issue, not blocking.

## Files Changed

- `src/grid_tactics/enums.py` — Added `ActionType.DECLINE_POST_MOVE_ATTACK = 8` (appended at end for numeric stability).
- `src/grid_tactics/game_state.py` — Added `pending_post_move_attacker_id: Optional[int] = None` to GameState frozen dataclass.
- `src/grid_tactics/action_resolver.py`
  - Deleted `_auto_attack_after_move` (60+ LOC dead code).
  - `_apply_move` no longer auto-attacks. For melee (`attack_range == 0`) with at least one in-range enemy from the new tile, sets `pending_post_move_attacker_id`. Ranged minions never enter pending.
  - Added `_has_any_attack_target` helper.
  - `resolve_action` now gates on pending state: only ATTACK (with the pending attacker) or DECLINE_POST_MOVE_ATTACK is legal; everything else raises ValueError. React window fires AFTER pending is cleared (move+attack OR move+decline = one window). DECLINE outside pending state is illegal.
- `tests/test_action_resolver.py` — Added `TestPendingPostMoveAttack` class with 8 new tests covering full lifecycle.
