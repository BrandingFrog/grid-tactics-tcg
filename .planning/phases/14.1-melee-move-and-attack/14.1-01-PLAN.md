<encoding_decision>
**Action-space layout [0:1262] is PRESERVED.** No new slots are added.

Encoding strategy:
1. Add a new `ActionType.DECLINE_POST_MOVE_ATTACK = 8` enum value (Python only — does NOT get its own action-space slot; it is encoded by reusing the PASS slot [1001] when `state.pending_post_move_attacker_id is not None`. In pending state, slot 1001 means "decline post-move attack" instead of "pass turn". In pending state, PASS-as-end-turn is illegal — the player must either attack or decline.)
2. Post-move ATTACK reuses the existing ATTACK[350:975] slots, but legal_actions filters them so only in-range targets from the moved attacker are legal while pending state is set.
3. While `pending_post_move_attacker_id is not None`: only ATTACK slots from that attacker AND slot 1001 (decline) are legal. All other slots (PLAY_CARD, MOVE, SACRIFICE, normal PASS, DRAW) are masked out.
4. Rationale: Zero RL-checkpoint invalidation. Existing checkpoints become stale (they learned the old auto-attack) but are loadable. Note as known issue, not blocking.

Tensor parity: `pending_post_move_attacker_id` becomes a `[N]` int tensor in tensor_engine state with sentinel -1 for "no pending". Mirrored in Wave 2.
</encoding_decision>
