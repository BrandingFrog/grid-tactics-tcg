# Phase 13 — UI Design Contract

> Visual and interaction contract for the Board & Hand UI phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/JS) |
| Icon library | none (text labels and Unicode symbols only) |
| Font | LuckiestGuy (Google Fonts CDN) for card names/stats; Inter (system-ui fallback) for UI chrome |

**Source:** D-05/D-06 mandate LuckiestGuy per YGO reference. Dashboard uses Inter/system-ui. No build step, no framework — consistent with existing dashboard pattern.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Grid cell gaps, inline icon padding, attribute circle border |
| sm | 8px | Hand card gap, compact stat padding, log entry padding |
| md | 12px | Board padding, card section padding, button gap (matches dashboard .card padding=16 but board is denser) |
| lg | 16px | Section padding (stat bars, room bar, opponent/self info bars) |
| xl | 24px | Lobby section margins, deck builder section breaks |
| 2xl | 32px | Page-level content padding |
| 3xl | 60px | Lobby vertical centering padding |

Exceptions:
- Board cell minimum height: 68px (matches existing `.duel-cell` min-height)
- Touch/click target minimum: 44px for all interactive buttons (deck builder add/remove, lobby buttons)

**Source:** Dashboard uses 4/8/12/16/20px increments. Board density demands tighter spacing (4px cell gap) than typical UI.

## Typography

### UI Chrome (Inter / system-ui)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | 400 | 1.5 | Log entries, descriptions, tooltip text |
| Label | 10px | 700 | 1.2 | Section headers (uppercase, 1.5px letter-spacing), stat labels ("HP", "MANA") |
| Heading | 14px | 700 | 1.2 | Player names, section titles |
| Display | 18px-28px | 900 | 1.0 | Stat values (HP/Mana numbers), room code, lobby heading |

### Card Text (LuckiestGuy)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Card Name (hand) | 14px | 400 | 1.1 | Full-size hand card name, white with 1px black text-stroke |
| Card Stats (hand) | 12px | 400 | 1.0 | ATK/HP labels on hand cards, white with 1px black text-stroke |
| Card Mana (hand) | 14px | 400 | 1.0 | Mana cost badge number |
| Card Name (board) | 9px | 700 | 1.1 | Compact minion name on board (truncated with ellipsis) |
| Card Stats (board) | 11px | 800 | 1.0 | ATK/HP on board minions |

**Note:** LuckiestGuy only comes in weight 400. "Bold" appearance is achieved via text-stroke and the font's inherent thick letterforms. Board minion names use Inter at 9px/700 for readability at small sizes.

**Source:** D-05/D-06 (LuckiestGuy from YGO reference). Dashboard uses 9-26px range with Inter. Font import: `@import url('https://fonts.googleapis.com/css2?family=Luckiest+Guy&display=swap');`

### Card Frame Anatomy (Full / Hand variant)

```
+--[Mana Badge]------------------+
|  [3]                           |
|  +-------------------------+   |
|  |                   [Attr]|   |
|  |   Art Area        [  O ]|   |
|  |   (type-tinted)        |   |
|  +-------------------------+   |
|      Card Name                 |
|  ATK/20            HP/10       |
+---------------------------------+
```

- Mana badge: 16% width circle, top 3%, left 3%, blue `rgb(30,100,220)`, white text, 2px dark border, z-index 2
- Art area: 96% width, 1:1 aspect, 2% top margin, 15% white overlay on type color, 1px dark border, 3% radius
- Attribute circle: 18% of art width, top-right 4%/4%, element color fill, 2px dark border, 50% radius. Inner ring: 82% size, 1px white semi-transparent border
- Card name: centered, white, 1px black text-stroke, `paint-order: stroke fill`, LuckiestGuy, ellipsis overflow
- Stats row: flex space-between, 4% horizontal padding, white with 1px black text-stroke

**Source:** D-05, D-06, RESEARCH.md Pattern 3. Translated from `CardFrameBuilder.luau`.

## Layout Specifications

### Game Screen Layout

```
+--[Header: 52px]---------------------------------------------+
|  GRID TACTICS  |  Overview  Training  Cards  Games  Duel     |
+--------------------------------------------------------------+
|                                                              |
|  +--[Main: flex-grow]---------------+  +--[Sidebar: 260px]-+|
|  | [Room Bar]                       |  | [Game Log]        ||
|  | [Opponent Info]                  |  |                   ||
|  | [Board 5x5]                     |  |                   ||
|  | [Self Info]                      |  |                   ||
|  | [Hand]                           |  | [Leave Button]    ||
|  +----------------------------------+  +-------------------+|
|                                                              |
+--------------------------------------------------------------+
```

- Max-width: 1100px, centered (`margin: 0 auto`)
- Layout: CSS Grid `grid-template-columns: 1fr 260px`, 16px gap
- At `<=900px`: single column (sidebar below main)
- Board container: 12px padding, full width of main column
- Board cells: equal 1fr columns, 4px gap

### Board Grid

```
CSS: display: grid; grid-template-columns: repeat(5, 1fr); gap: 4px;
```

- 5 columns, equal width
- 5 rows rendered by JS (display order depends on player perspective per D-03)
- Each cell: `aspect-ratio: 1`, card2 bg, border, 8px radius
- No row/column labels in Phase 13 (keep minimal)

### Hand Layout

```
CSS: display: flex; gap: 8px; overflow-x: auto; padding: 8px 0;
```

- Horizontal scroll when hand exceeds container width
- Cards: 120px width, fixed (not responsive)
- Padding: 8px vertical for hover translateY headroom
- Scroll: native horizontal scroll with `-webkit-scrollbar` styled to match theme

## Interaction States (Read-Only Phase)

Phase 13 is read-only rendering. No action submission occurs. Click interactions are Phase 14.

| Element | States |
|---------|--------|
| Hand card | default, hover (translateY -6px, cyan border glow), dimmed (unaffordable) |
| Board cell | default, hover (cyan border, darker bg) |
| Board minion | static display only (owner tint, attribute, stats) |
| Lobby button | default, hover (translateY -1px, shadow), disabled (opacity 0.3) |
| Turn banner | your-turn (green pulse animation), opponent-turn (muted, no animation) |
| Phase badge | action (cyan bg, dark text), react (yellow bg, dark text) |
| Deck builder card | default, hover (scale 1.02), selected (green border, count badge updates) |

### Animations

| Animation | CSS | Duration | Usage |
|-----------|-----|----------|-------|
| Turn pulse | `@keyframes pulse { 0%,100% { opacity:1 } 50% { opacity:0.4 } }` | 1.5s infinite | Green dot next to "YOUR TURN" |
| Card hover lift | `transform: translateY(-6px); box-shadow: 0 8px 24px rgba(0,212,255,.2)` | 150ms ease | Hand cards on hover |
| Cell hover | `border-color: var(--cyan); background: #1e1e40` | 150ms ease | Board cells on hover |
| Button hover lift | `transform: translateY(-1px); box-shadow: 0 4px 16px rgba(0,0,0,.3)` | 150ms ease | All buttons |

## Data Flow Contract

### Events Consumed (from server)

| Event | Data Shape | Triggers |
|-------|-----------|----------|
| `game_start` | `{ card_defs, state, your_player_idx, legal_actions }` | Initial render of board, hand, stats |
| `state_update` | `{ state, legal_actions }` | Re-render board, hand, stats, turn indicator |
| `game_over` | `{ winner, reason }` | Phase 14 scope (not rendered in Phase 13) |
| `error` | `{ message }` | Display error toast/message |

### Client State Variables

| Variable | Type | Source |
|----------|------|--------|
| `cardDefs` | `Object<numericId, CardInfo>` | `game_start.card_defs` |
| `gameState` | `Object` | `game_start.state` / `state_update.state` |
| `myPlayerIdx` | `0 or 1` | `game_start.your_player_idx` |
| `legalActions` | `Array` | `game_start.legal_actions` / `state_update.legal_actions` |

### Card Info Shape (from enhanced card_defs)

```javascript
{
  card_id: "minion_fire_imp",    // string identifier
  name: "Fire Imp",              // display name
  card_type: 0,                  // 0=Minion, 1=Magic, 2=React
  mana_cost: 2,                  // integer
  attack: 20,                    // integer or null
  health: 10,                    // integer or null
  attack_range: 0,               // integer
  element: 1,                    // 0-6 or null
  tribe: "Imp",                  // string or null
  effects: [{ type, trigger, target, amount }],  // array
  react_condition: null           // integer or null
}
```

### Minion State Shape (from state.minions[])

```javascript
{
  position: [row, col],          // [int, int]
  owner: 0,                      // 0 or 1
  card_numeric_id: 5,            // maps to cardDefs key
  current_health: 10,            // current HP
  attack_bonus: 0,               // added to base attack
  effects_applied: []            // active buffs/debuffs
}
```
